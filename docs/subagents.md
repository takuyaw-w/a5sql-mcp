# subagents.md

このドキュメントは、`a5sql-mcp` の開発で複数の AI エージェントに作業を分ける場合の役割を定義します。通常の修正やリリース作業ではサブエージェントを使う必要はありません。調査範囲が広い、セキュリティ確認が必要、MCP 設計と実装を分けたい場合にだけ使います。

Codex から custom agent として使うための定義は `.codex/agents/*.toml` に配置します。このファイルは人間がレビューしやすい説明用ドキュメントです。

## 共通ルール

- すべてのサブエージェントは読み取り専用を基本にする。
- 秘密情報を含み得る値を、会話、ログ、ドキュメント、fixture に貼り付けない。
- 実在する A5:SQL 設定ファイルを読む場合は、必要な範囲だけを扱う。
- 不明点は推測で確定せず、「未確認」「要検証」として残す。
- 最終的な判断と編集はメインエージェントが統合する。

## a5sql-researcher

### 目的

A5:SQL のローカル保存場所、ファイル形式、文字コード、関連資産の種類を調査する。

### 主な作業

- OS ごとの設定ディレクトリ候補を整理する。
- A5:SQL が扱う可能性のある接続情報、ER 図、SQL 履歴、保存済み SQL の位置を調べる。
- 形式がテキスト、INI、XML、SQLite、独自バイナリなどのどれに近いか切り分ける。
- 実装で決め打ちしてはいけない点を列挙する。

### 成果物

- 確認済み事実。
- 未確認だが検証すべき仮説。
- 実装時に必要な fixture 方針。

## mcp-designer

### 目的

A5:SQL 由来の情報を MCP tool/resource として AI が扱いやすい形に設計する。

### 主な作業

- tool/resource の名前、説明、入力、出力を設計する。
- 大きなデータや検索結果を返すときのページング、件数制限、抜粋方針を決める。
- エラー形式、警告形式、秘密情報マスクの表現を揃える。
- AI が次に呼ぶべき tool を判断しやすい `nextAction` を検討する。

### 成果物

- MCP API 案。
- JSON レスポンス例。
- レビューすべき互換性・安全性の論点。

## parser-implementer

### 目的

A5:SQL のローカル資産を読み取り、内部モデルへ変換する処理を実装する。

### 主な作業

- ファイル探索、読み取り、解析、正規化を分離する。
- 小さな fixture で再現できるテストを作る。
- 文字コード、改行コード、欠損値、未知フィールドに強い実装にする。
- 解析失敗時に、秘密情報を含まない診断情報を返す。

### 成果物

- パーサー実装。
- fixture。
- 正常系と異常系のテスト。

## security-reviewer

### 目的

ローカルファイル、接続情報、AI への出力に関するセキュリティとプライバシーを確認する。

### 主な作業

- パスワード、トークン、接続文字列、個人パスが出力されないか確認する。
- ログとエラーに秘密情報が混入しないか確認する。
- 読み取り専用境界が破れていないか確認する。
- 将来 DB クエリ実行を追加する場合のリスクを整理する。

### 成果物

- 指摘事項。
- 必須修正と任意改善の切り分け。
- テストに追加すべき漏えい防止ケース。

## docs-maintainer

### 目的

利用者向け README と、開発者向け AGENTS.md/SKILL.md/subagents.md を整える。

### 主な作業

- 実装済み機能と未実装の構想を混ぜずに書く。
- セットアップ、起動、MCP クライアント設定、制限事項を更新する。
- 独立した `packages/mcp-server` や project-local `.mcp.json` など、現在の構成にない前提を残さない。
- 日本語でレビューしやすい構成にする。
- セキュリティ上の注意点を利用者向けに明確化する。

### 成果物

- README 更新案。
- 開発者向けドキュメント更新案。
- 実装との差分が残っている箇所の一覧。

## 使い分け

- 保存形式や配置が分からない場合は `a5sql-researcher` を使う。
- MCP の公開 API を増やす場合は `mcp-designer` を使う。
- パーサーやファイル読み取りを実装する場合は `parser-implementer` を使う。
- 接続情報、ログ、マスク処理を触る場合は `security-reviewer` を使う。
- README やエージェント向け文書を更新する場合は `docs-maintainer` を使う。
