# a5sql-mcp 0.9.10 Preflight Contract Audit Design

## Goal

`0.9.10: Preflight Contract Audit` として、0.9.9 で凍結した MCP API surface が実装、README、AGENTS.md、repo-local skill、test で一致していることを確認できる状態にする。

0.9.10 は新機能追加ではなく、1.0.0 前の contract drift 検出を目的とする。runtime behavior は原則変更せず、説明不能なズレが見つかった場合だけ 1.0.0 contract としてどちらへ寄せるかを明示して最小修正する。

## Background

現在の実装 version は root workspace、`packages/parser`、`packages/core`、`packages/cli`、`A5SQL_MCP_SERVER_VERSION`、version assertion test がすべて `0.9.9` のままである。`ROADMAP.md` では 0.9.10 が `Preflight Contract Audit` として定義されているが、実装 milestone としては未達成である。

0.9.9 では `.a5er` 起動時の `tools/list` を基準に tool 名、description、input schema、stable read-only / experimental draft の分類を固定した。0.9.10 では、その凍結内容が docs と tests に継続して反映されているかを release 前に再監査する。

## Scope

### In Scope

- `.a5er` 起動時の `tools/list` を source of truth として、tool 名、description、input schema が維持されていることを test で確認する。
- stable read-only tool と experimental draft tool の分類が、server registration、README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md`、test で一致していることを確認する。
- docs / test に軽量な drift guard を追加または強化する。
- package versions と `A5SQL_MCP_SERVER_VERSION` を `0.9.10` に lockstep で更新する。
- 0.9.10 の検証として `rtk pnpm release:check` と `rtk pnpm published:check` を実行する。

### Out of Scope

- 新しい MCP tool / resource の追加。
- DB 接続、SQL 実行、A5:SQL 設定書き込み、資格情報の復号や表示。
- 0.9.11 の範囲である representative `structuredContent` の trusted metadata / untrusted payload 境界の横断監査。
- 0.9.12 の範囲である installed package / MCP client matrix の拡張。
- 1.0.0 tag / publish 作業。

## Design

0.9.10 は feature branch または worktree で実施する。implementation plan の Task 0 で `rtk pnpm agent:preflight` を実行し、`main` 上で直接実装しない。

中心となる変更は test の contract guard である。既存の `packages/cli/test/mcp-server-smoke.test.ts` には 0.9.9 API freeze test があるため、これを 0.9.10 の preflight audit として強化する。`.a5er` 起動時の tool 一覧を取得し、期待 tool 名、description の有無、input schema の存在、experimental draft marker の有無を確認する。

docs 側は既存の `packages/cli/test/docs-redaction.test.ts` を使い、README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` が 0.9.9 API Freeze の分類だけでなく、0.9.10 Preflight Contract Audit として drift guard の目的を説明していることを確認する。文言は利用者向け README と agent 向け AGENTS / SKILL で役割に合わせて短くし、同じ tool 分類を別解釈しないようにする。

version は root workspace、parser、core、cli、MCP server metadata、version assertion test を `0.9.10` に揃える。個別 package の version をずらさない。

## Contract Rules

0.9.10 の contract source of truth は `.a5er` 起動時の `tools/list` とする。

stable read-only tool は `experimental draft tool` marker を持たない。`review_a5sql_schema`、`suggest_schema_changes`、`compare_a5er_with_live_schema` は stable read-only の分析・比較 tool として扱う。

experimental draft tool は description に `experimental draft tool` marker を持つ。対象は `generate_sql_select`、`generate_mermaid_er_diagram`、`generate_model_files`、`generate_schema_markdown`、`generate_migration_plan` のみとする。

全 tool はローカルファイル読み取り専用であり、DB へ接続せず、SQL を実行せず、ファイルシステムへ生成物を書き込まない。0.9.10 ではこの境界を docs と test で確認するが、新しい実行機能は追加しない。

## Error Handling

preflight audit で drift が見つかった場合は、test failure の内容を先に確認する。docs だけが古い場合は docs を source of truth に合わせる。server registration が docs / test と違う場合は、1.0.0 contract として維持すべき分類を明示し、最小の registration または test 修正に限定する。

`warnings`、`message`、`code`、`nextAction` に A5:SQL 由来 payload が混ざらないことの横断監査は 0.9.11 で扱う。0.9.10 では API surface と tool classification の drift を主対象にする。

## Testing

TDD で進める。最初に version assertion と contract audit assertion を `0.9.10` 前提に更新または追加し、現状 `0.9.9` のため失敗することを確認する。

focused test は次を想定する。

- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-server-smoke.test.ts`
- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- docs-redaction.test.ts`

最終検証は次を実行する。

- `rtk pnpm release:check`
- `rtk pnpm published:check`

## Acceptance Criteria

- `A5SQL_MCP_SERVER_VERSION` と全 package version が `0.9.10` に揃っている。
- `.a5er` 起動時の `tools/list` に、期待する stable read-only tool と experimental draft tool だけが含まれている。
- experimental draft tool だけが `experimental draft tool` marker を description に持つ。
- README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md`、server registration、test の tool 分類に説明不能なズレがない。
- `rtk pnpm release:check` と `rtk pnpm published:check` が成功する。
