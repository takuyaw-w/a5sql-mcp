# a5sql-mcp 0.10.1 Scoped Tool Surface / Client Profile Design

## 背景

`ROADMAP.md` の 0.10.1 は、20 個を超える MCP tool を常に同じ `tools/list` に出す負荷を下げるため、互換性を保った optional profile を追加する milestone である。

現在の MCP server は `a5sql-mcp --mcp <file>` で起動し、起動時ファイルが `.a5er` の場合は file / asset tool に加えて schema exploration、analysis、draft generation tool をまとめて登録する。`.sql` / text の場合は `.a5er` 専用 tool を登録しない。

0.10.1 では、この既存挙動を default として維持する。profile は tool を hidden にするだけであり、root 境界、秘密情報マスク、untrusted content、draft disclosure、DB 非接続、SQL 非実行、ファイル非書き込みの contract は変更しない。

## 目的

- 既定の `--mcp <file>` 起動では 1.0.0 互換の tool 一覧を維持する。
- `--tool-profile <profile>` により、MCP client が用途に近い tool surface を選べるようにする。
- `core-read`、`schema-explore`、`draft-generation`、`all` の profile を提供する。
- profile ごとの `tools/list` E2E を追加し、default 互換と tool 数削減を同時に確認する。
- README に profile の使い分けと client 設定例を追加する。

## 非目的

- tool 名、input schema、主要 output field の変更。
- tool handler の挙動変更。
- 生成補助 tool から `experimental draft tool` marker を外すこと。draft graduation は 0.11.0 以降の範囲とする。
- MCP Resource の追加。Resource Gateway pilot は 0.10.3 の範囲とする。
- DB 接続、SQL 実行、A5:SQL 設定や ER 図ファイルへの書き込み、資格情報の復号・表示。
- runtime tool による profile 切り替えや session state の導入。

## 設計

### Profile 指定

CLI に `--tool-profile <profile>` を追加する。

```bash
a5sql-mcp --mcp /absolute/path/to/model.a5er --tool-profile schema-explore
```

指定できる値は次の 4 つに限定する。

- `all`
- `core-read`
- `schema-explore`
- `draft-generation`

profile 未指定時は `all` と同じ挙動にする。これは既存の README、Codex、Cursor、Claude Code の設定例を壊さないための互換性条件である。

不正な profile 値は server 起動前に CLI error として扱う。stderr に許可値を示し、stdout は MCP JSON-RPC 用に使わない。

### Profile の tool 集合

`all` は現在の挙動を維持する。起動時ファイルが `.a5er` の場合は全 tool、`.sql` / text の場合は file / asset 系 tool のみを出す。

`core-read` は、起動時ファイルと明示 root 配下 asset を安全に読むための tool に絞る。

- `describe_a5sql_file`
- `parse_a5sql_file`
- `read_a5sql_file`
- `detect_a5sql_locations`
- `read_a5sql_asset`
- `list_a5sql_connections`
- `search_a5sql_assets`
- `parse_a5sql_asset`

`schema-explore` は `core-read` に加えて、`.a5er` の構造を調べる tool を出す。

- `list_a5sql_tables`
- `describe_a5sql_table`
- `explain_a5sql_table`
- `list_a5sql_relationships`
- `find_a5sql_tables`
- `find_a5sql_columns`

`draft-generation` は `core-read` に加えて、schema review、比較、review 用 draft 生成 tool を出す。

- `generate_sql_select`
- `generate_mermaid_er_diagram`
- `generate_model_files`
- `generate_schema_markdown`
- `review_a5sql_schema`
- `suggest_schema_changes`
- `compare_a5er_with_live_schema`
- `generate_migration_plan`

`.a5er` 以外で起動した場合、`.a5er` 専用 tool は profile に含まれていても登録しない。例えば text file に `--tool-profile schema-explore` を指定した場合は `core-read` 相当の tool だけが出る。この挙動は既存の「起動時ファイル種別で `.a5er` tool を出し分ける」contract を保つためである。

### 実装単位

新しい小さな module `packages/cli/src/mcp/tool-profiles.ts` を追加する。

この module は次を持つ。

- `ToolProfile` 型。
- `DEFAULT_TOOL_PROFILE`。
- profile 値の parser / validator。
- tool 名が profile に含まれるかを判定する関数。
- profile ごとの期待 tool 名配列を test から再利用できる export。

`packages/cli/src/index.ts` は `--mcp <file>` の後続引数として `--tool-profile <profile>` を読む。非 MCP の `a5sql-mcp <file>` parse mode には profile を適用しない。

`packages/cli/src/mcp/server.ts` は `McpServerOptions` に `toolProfile?: ToolProfile` を追加し、tool 登録前に profile 判定を行う。登録 handler、schema、description は変更しない。

### Docs

README に profile の短い節を追加する。

説明する内容:

- 何も指定しない場合は current / 1.0.0 compatible な `all`。
- `core-read` は asset 探索・範囲読み取り中心。
- `schema-explore` は `.a5er` のテーブル、カラム、リレーション確認中心。
- `draft-generation` は review 用 SQL / Mermaid / model / Markdown / migration 案や schema review 中心。
- profile は tool 表示を絞るだけで、安全境界や権限を広げない。

Codex / Cursor / Claude Code の既存例は壊さず、任意で `--tool-profile schema-explore` を足す例を 1 つ置く。

### Test

`packages/cli/test/mcp-server-smoke.test.ts` に profile E2E を追加する。

確認すること:

- profile 未指定の `.a5er` 起動では、現行の expected tool names と一致する。
- `all` は profile 未指定と一致する。
- `core-read` は core tool のみを返し、`.a5er` 専用 tool と draft tool を含まない。
- `schema-explore` は core tool と schema exploration tool を返し、draft generation tool を含まない。
- `draft-generation` は core tool と review / comparison / draft generation tool を返し、schema exploration だけの tool を含まない。
- `.sql` または text 起動で `schema-explore` を指定しても、`.a5er` 専用 tool は出ない。

`packages/cli/scripts/published-check.mjs` は default 起動の互換性を維持しつつ、軽量に `--tool-profile core-read` または `--tool-profile schema-explore` の `tools/list` を確認する。これにより installed package の CLI option parse と MCP startup を同時に検証する。

`packages/cli/test/docs-redaction.test.ts` は public guidance files に profile の説明があり、`--tool-profile`、`core-read`、`schema-explore`、`draft-generation`、`all` が記載されていることを確認する。

## バージョンと ROADMAP

0.10.1 の実装として、root workspace、parser、core、cli、MCP server metadata、smoke test expectation の version を `0.10.1` に揃える。

実装完了時に `ROADMAP.md` の `0.10.1` を `[x]` に変更する。0.10.2 以降は未完了のまま残す。

## 検証

- `rtk pnpm agent:preflight`
- profile parser / focused MCP smoke tests
- docs drift guard
- `rtk pnpm release:check`
- `rtk pnpm published:check`

  0.10.1 は MCP startup option と `tools/list` の contract に関わるため、local source tree だけでなく installed package 経由の profile 指定も確認する。

## リスク

- default tool 一覧を変えると 1.0.0 互換性を壊す。profile 未指定は必ず `all` として扱う。
- profile 名や tool 集合を増やしすぎると、かえって client 設定が難しくなる。0.10.1 では ROADMAP にある 4 profile だけに限定する。
- `schema-explore` や `draft-generation` を `.sql` / text 起動でもエラーにすると、既存のファイル種別による tool 出し分けと相性が悪い。`.a5er` 専用 tool は単に登録しない。
- profile を権限機構として説明すると誤解を招く。profile は tool visibility の調整であり、root boundary や secret masking の代替ではない。

## 承認事項

この design は `--tool-profile <profile>` による CLI option 方式で進める。profile 未指定時は現行互換の `all` とし、profile は tool visibility を絞るだけで安全境界や handler 挙動を変えない。
