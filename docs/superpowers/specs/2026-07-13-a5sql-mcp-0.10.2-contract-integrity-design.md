# a5sql-mcp 0.10.2 Contract Integrity, Structured Errors and Safe Observability Design

Status: approved

## 背景

`ROADMAP.md` の 0.10.2 は、既存 22 tool の安全境界、bounded read、partial parse、parser fidelity、structured error、observability を 1.0.0 前に修正する release-blocking milestone である。

現行実装には、test、`release:check`、`published:check` が成功していても検出できない次の contract gap がある。

- `warnings` を trusted metadata と宣言しながら、A5:ER、live schema、tool input 由来の可変文字列を埋め込む経路がある。
- `list_a5sql_connections` が A5:SQL 設定由来 payload を返しながら、untrusted content contract を通っていない。
- `read_a5sql_asset` の source truncation と asset lookup cutoff が `parse_a5sql_asset` まで伝播しない。
- truncated `.a5er` を完全 schema のように解析できてしまう。
- SQL parser が内部で 100 statement に固定され、MCP の `maxStatements: 1000` と一致しない。
- A5:ER の scalar を一律に JavaScript `Number` へ変換し、BIGINT や高精度 decimal を壊し得る。
- `read_a5sql_asset` が masking 補助のために最初の `maxBytes` を超えて再読込する。
- assetId lookup に file-count 上限がない。
- tool call observability は transport error 以外に存在せず、現行 transport error は生 message を stderr へ出す。

## 目的

- trusted guidance と untrusted detail を機械的に区別できる contract にする。
- source read と asset lookup の上限・打ち切り理由を core から MCP response まで伝播する。
- truncated `.a5er` を完全 schema として返さない。
- SQL statement count と output limit を分離し、total / returned / truncation を正しく返す。
- A5:ER の schema fidelity に必要な数値文字列を壊さない。
- LLM client が retry、入力の絞り込み、user escalation、停止を判断できる structured error を返す。
- stdout を JSON-RPC 専用に保ったまま、秘密情報を含まない opt-in observability を提供する。

## Architecture 上の位置づけ

Primary pattern は read-only Resource Gateway とする。A5:ER / SQL / 接続候補 / live schema snapshot を LLM 向けに変換する Domain-Specific Adapter の性質を併せ持つ。

0.10.2 では次の境界を変更しない。

- DB に接続しない。
- SQL を実行しない。
- ファイルシステムへ書き込まない。
- `roots` / `A5SQL_MCP_ROOTS` の明示境界を維持する。
- password、token、connection string、秘密鍵を返さない。
- A5:SQL 由来本文、コメント、識別子、SQL、draft を untrusted content として扱う。

## 設計

### 1. Trusted diagnostics と untrusted details

`warnings`、`message`、`code`、`nextAction` は固定 code / guidance のみを含む trusted metadata とする。A5:ER、live schema、tool input 由来の値は `warningDetails` に分離する。

例:

```json
{
  "warnings": ["a5er_encoding_mismatch"],
  "warningDetails": [
    {
      "code": "a5er_encoding_mismatch",
      "declaredEncoding": "UTF8",
      "decodedEncoding": "shift_jis"
    }
  ],
  "trustedMetadataFields": ["warnings"],
  "untrustedPayloadFields": ["warningDetails"]
}
```

設計ルール:

- `warnings` は固定 code の配列にする。
- table / column / relationship / schema 名、encoding、tool input、count detail を文字列連結して `warnings` へ入れない。
- 可変値を含む `warningDetails` は `untrustedPayloadFields` に分類する。
- `content[0].text` は `structuredContent` と同じ JSON shape を維持する。
- generation tool の `draftOutputFields` と `draftIsDerivedFromUntrustedInput` は維持する。

### 2. Connection payload boundary

`list_a5sql_connections` の success / error response を `withUntrustedPayloadContract()` へ統一し、`connections` を `untrustedPayloadFields` に追加する。

connection name、host、database、user、source path、matched key は、mask 済みであっても A5:SQL 設定由来 payload として扱う。password、token、connection string を返さない既存 contract は維持する。

`totalConnectionCount` の意味修正は 0.10.3 の範囲に残し、0.10.2 では安全分類だけを修正する。

### 3. Bounded asset lookup と read metadata

core に metadata 付き asset lookup API を追加し、既存 `readA5sqlAsset()` は compatibility wrapper として残す。

metadata は次を含む。

- `sourceSizeBytes`
- `bytesRead`
- `sourceTruncated`
- `visitedFileCount`
- `lookupTruncated`
- `cutoffReason`
- `maxFiles`

assetId lookup は既定 5,000 files、最大 100,000 files で停止する。上限到達時は通常の not-found と区別し、`asset_lookup_truncated` を返す。

direct path read は directory scan を行わず、既存の root containment check を維持する。

### 4. Partial parse policy

ファイル種別ごとに source truncation の扱いを分ける。

- `.a5er`: 不完全な schema を完全 schema として返さない。parser を実行せず、`source_truncated` structured error を返す。
- `.sql`: bounded partial parse を許可する。`sourceTruncated` と `trailingStatementMayBeIncomplete` を返し、source 全体の確定 total ではないことを示す。
- text: 既存の範囲読み取りを維持し、source truncation metadata を返す。

### 5. SQL statement metadata

SQL 分割処理と返却件数制限を分離する。

metadata 付き SQL parse API は次を返す。

- `statements`
- `totalStatementCount`
- `returnedStatementCount`
- `statementsTruncated`
- `trailingStatementMayBeIncomplete`

`maxStatements` は保持・返却する statement 数だけを制限する。隠れた 100 件制限を削除し、既存 `parseSqlStatements()` は compatibility API として残す。

### 6. A5:ER scalar fidelity

`parseScalar()` は `Number.isSafeInteger()` を満たす整数だけ number 化する。

- safe integer: number
- unsafe integer: string
- decimal: string
- position、relation type、key order など明示的に数値が必要な構造 field: 既存 `toNumber()` で変換

これにより、`9007199254740993` が `9007199254740992` に変化する問題を防ぐ。

### 7. Read budget の一貫性

`read_a5sql_asset` は core で最初に読んだ bounded content だけを masking に使う。masking 補助を理由に別上限で再読込しない。

### 8. Structured error

代表 error を次の shape に揃える。

```json
{
  "code": "fixed_error_code",
  "message": "固定された説明",
  "nextAction": "固定された次の行動",
  "retryable": false
}
```

対象:

- `roots_required`
- `asset_not_found`
- `asset_lookup_truncated`
- `source_truncated`
- `configured_file_is_not_a5er`
- `unsupported_asset_type`
- handler 内の予期しない例外

予期しない例外では、生 message、絶対 path、入力値を response へ含めない。

### 9. Safe observability

既定では tool call log を出さない。`A5SQL_MCP_OBSERVABILITY=stderr` を明示した場合だけ、stderr に 1 行 JSON を出す。stdout は JSON-RPC 専用のまま維持する。

記録する値:

- tool name
- process 内で相関可能な input hash
- latency
- output size
- fixed error code
- success / failure

記録しない値:

- tool input の値
- A5:SQL 本文、SQL 本文
- table / column 名
- assetId
- root、絶対 path
- password、token、connection string、秘密鍵
- 生の例外 message

input hash は起動時に生成する random key を使った HMAC とし、同一 process 内だけで相関できるようにする。transport error も生 message を出さず、固定 code と error 種別に限定する。

## Test 方針

TDD で次を固定する。

- prompt injection 風 table / column / live schema / tool input が trusted field に入らない。
- connection payload が untrusted として分類される。
- truncated `.a5er` が完全 schema として返らない。
- truncated SQL が partial であることを示す。
- 101 / 1,000 / 1,001 statement の count が正しい。
- BIGINT と高精度 decimal が壊れない。
- `maxBytes` を超える再読込をしない。
- asset lookup が file-count 上限で停止し、cutoff metadata を返す。
- observability log に秘密値、SQL 本文、絶対 path が含まれない。
- `content[0].text` と `structuredContent` が一致する。
- MCP client と installed package bin 経由でも同じ contract を確認できる。

## 非目的

以下は 0.10.3 に残す。

- CLI と core decoder の統合。
- Windows / Linux の path case 処理。
- model generator の identifier escape。
- MCP `outputSchema`。
- connection total count semantics の変更。
- CLI 直実行時の masking policy 変更。
- PR / main CI と Windows matrix。
- publish workflow 自体の変更。
- tool 数、profile、既定 tool surface の変更。

## Version、docs、完了条件

- README、AGENTS.md、repo-local skill の安全境界を同期する。

## 承認事項

この design は、fixed warning codes + untrusted `warningDetails`、bounded asset lookup metadata、`.a5er` の fail-closed partial parse、SQL partial metadata、safe scalar conversion、opt-in HMAC observability の方針で実装する。
