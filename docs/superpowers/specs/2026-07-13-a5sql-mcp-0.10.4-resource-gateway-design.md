# a5sql-mcp 0.10.4 MCP Resource Gateway Pilot Design

Status: approved

## 背景

`ROADMAP.md` の 0.10.4 は、既存 MCP tool に加えて、安全な read-only summary を MCP Resource として公開する価値があるかを検証する pilot である。

現行 server は `packages/cli` の stdio MCP mode で 22 tool を公開しているが、Resource は未登録である。起動時ファイルの metadata や A5:ER schema の bounded summary は Resource と相性がよい一方、asset root の検索には明示引数が必要であり、root や path を Resource URI に埋め込むと path privacy と root boundary を弱める。

0.10.4 は Resource Gateway / Domain-Specific Adapter という既存の architecture classification を、実際の MCP Resource surface で検証する。既存 tool の名前、input schema、output contract、tool profile semantics は変更しない。

## 目的

- 起動時ファイルの安全な metadata summary を固定 URI の Resource として公開する。
- `.a5er` 起動時だけ、bounded schema summary を固定 URI の Resource として公開する。
- `resources/list` と `resources/read` を MCP client E2E と installed-package validation で固定する。
- Resource URI、metadata、content からユーザー固有 path と秘密情報が漏れないことを保証する。
- A5:SQL 由来 payload を tool と同じ untrusted content contract で扱う。
- Resource 化に適さない asset search は tool-only のまま残し、その理由を docs に記録する。

## Non-goals

- `search_a5sql_assets`、`read_a5sql_asset`、`parse_a5sql_asset` の Resource 化。
- root、path、asset ID を含む Resource URI template。
- 起動時ファイル本文や SQL 全量の Resource 配信。
- Resource subscription、list-changed notification、server-side session state。
- `--tool-profile` による Resource filtering。
- DB 接続、SQL 実行、ファイル書き込み、資格情報の復号・表示。
- 既存 22 tool の削除、rename、input/output contract 変更。

## Architecture 上の位置づけ

Primary pattern は read-only Resource Gateway、cross-cutting attribute は Domain-Specific Adapter とする。

Resource Gateway として、Resource は backend file を直接 expose せず、固定 URI、bounded summary、secret masking、untrusted payload label を持つ LLM-facing contract に変換する。Domain-Specific Adapter として、A5:ER の parse status、table/relationship count、warning metadata を安定した JSON に整形する。

次の anti-pattern を避ける。

- Unsanitized Resource Content: table 名、column 名、comment、relationship、warning detail を trusted instruction として返さない。
- God Resource: metadata、全文、検索、解析、生成を一つの Resource にまとめない。
- Synchronous Long-Running Operation: root scan や全文 parse を Resource read の暗黙処理にしない。
- Vague Resource Description: Resource が summary であり、本文・DB・書き込みを扱わないことを description に明記する。

## Resource surface

### 1. Configured file summary

- Name: `configured-file-summary`
- URI: `a5sql://configured-file/summary`
- MIME type: `application/json`
- 登録条件: 起動時ファイルの kind に関係なく常に登録

主な payload:

- `schemaVersion`
- `resultType: "configured_file_summary_resource"`
- `kind`
- `encoding`
- `sizeBytes`
- `modifiedAt`
- `readOnly: true`
- `writesToFileSystem: false`
- `connectsToDatabase: false`
- `executesSql: false`
- `contentIsUntrusted: false`

絶対 path、file URI、basename、本文、抜粋は含めない。`resources/list` の Resource metadata にも size や lastModified を埋め込まず、read 時点の値を content で返す。

### 2. Configured schema summary

- Name: `configured-schema-summary`
- URI: `a5sql://configured-file/schema-summary`
- MIME type: `application/json`
- 登録条件: 起動時ファイルが `.a5er` の場合だけ登録

主な payload:

- `schemaVersion`
- `resultType: "configured_schema_summary_resource"`
- `kind: "a5er"`
- `parseStatus`
- `formatVersion`
- `fileEncoding`
- `a5erEncoding`
- `tableCount`
- `relationshipCount`
- `returnedTableCount`
- `returnedRelationshipCount`
- `tables`
- `relationships`
- `truncated`
- `warnings`
- `warningDetails`
- `contentIsUntrusted: true`
- `trustedMetadataFields`
- `sourceMetadataFields`
- `untrustedPayloadFields`
- `nextAction`

table と relationship は固定上限で返す。pilot の default は各 20 件とし、URI から上限を変更できないようにする。より深い探索や任意の paging は既存 `list_a5sql_tables`、`find_a5sql_tables`、`describe_a5sql_table`、`list_a5sql_relationships` を使わせる。

`tables`、`relationships`、`warningDetails` は A5:SQL 由来の未信頼 payload とする。`warnings`、`nextAction` は固定 guidance とし、A5:SQL 由来文字列を混ぜない。

## URI と path privacy

Resource URI は server 起動引数や filesystem path から生成しない。固定 URI は「現在の stdio server に設定されたファイル」を指す process-local identifier として扱う。

次を禁止する。

- `file://` URI。
- home directory、drive letter、root、basename を含む URI。
- root/path を query string、fragment、URI template variable に含めること。
- Resource title、description、content、error に absolute path を含めること。

asset search は `roots` / `A5SQL_MCP_ROOTS` を明示する必要がある。`resources/read` は tool input schema のような独立引数を持たないため、0.10.4 では tool-only を維持する。

## Secret masking と untrusted content

Resource JSON は serialization 前後の public-output boundary で既存 secret masking policy を適用する。tool handler 内に閉じている public masking helper は小さな共通 module に抽出し、既存 tool と Resource が同じ masking implementation を使う。

Resource regression fixture には次を含める。

- secret-like directory / file name。
- password、token、URL userinfo、ODBC credential を含む schema comment。
- prompt injection 風の table/comment payload。

`resources/list` と `resources/read` の serialized response に raw path/secret がないこと、未信頼 payload 自体は命令として扱われず `contentIsUntrusted` / `untrustedPayloadFields` で分類されることを確認する。

## Freshness と error contract

Resource read は既存 `getParsedFile` cache を使い、size / mtime が変わった場合は再 parse する。configured file summary は read 時点で `stat` を取り直す。

file が削除された、stat/read に失敗した、再 parse できない場合は、exception message や path を Resource response に流さない。固定 JSON error payload を返す。

- `resultType: "resource_error"`
- `code: "configured_file_unavailable"`
- 固定 `message`
- 固定 `nextAction`
- `retryable: true`

Resource protocol 自体の unknown URI handling は SDK に委ねる。

## Tool profile との関係

`--tool-profile` は 0.10.1 で導入した `tools/list` visibility filter であり、permission system ではない。0.10.4 Resource は profile に依存せず登録する。

これにより、profile semantics を変更せず、Resource surface を独立して検証できる。root boundary、masking、untrusted content、draft disclosure、DB/FS/SQL non-goals も変更しない。

## 実装構造

- `packages/cli/src/mcp/resources.ts`
  - Resource URI/name constants。
  - pure payload builder。
  - public JSON serialization。
  - fixed error payload。
  - `registerA5sqlResources()`。
- `packages/cli/src/mcp/public-output.ts`
  - tool handler から抽出する共通 masking helper。
- `packages/cli/src/mcp/server.ts`
  - server 作成時の Resource 登録だけを追加。

Resource の payload construction は pure function を中心にし、MCP callback、stat、parse cache と分離する。`server.ts` に payload shaping を追加しない。

## Test 方針

TDD で次を固定する。

- text / SQL / A5:ER 起動時の `resources/list`。
- `.a5er` だけ schema summary Resource がある。
- Resource URI/name/MIME type に path がない。
- `resources/read` の JSON payload shape。
- schema summary の 20 件上限と `truncated`。
- tool profile を変えても Resource 一覧が変わらない。
- file 更新後の read が新しい metadata/summary を返す。
- file disappearance が固定 error payloadになり、path/exception detail を漏らさない。
- table/comment の prompt injection payload が untrusted fields に分類される。
- raw secret、absolute path、URL credential、ODBC credential が response にない。
- existing tool response masking と 22 tool surface が変わらない。
- installed package bin から `resources/list`、`resources/read`、代表 tool call が成功する。

- A5:SQL 由来 payload が untrusted content として分類され、bounded summary だけが返る。
