# a5sql-mcp 0.9.3 Trusted Metadata / Untrusted Payload Design

## Goal

0.9.3 では、`nextAction`、`warnings`、`message`、`code` などの trusted metadata と、A5:SQL 由来の本文・識別子・コメントを含む untrusted payload の境界を、MCP tool 出力から機械的に読み取れるようにする。

## Scope

対象にするもの:

- A5:SQL 由来 payload を返す read / parse / search / schema exploration output。
- 生成補助 tool の draft output。
- prompt injection 文を含む synthetic fixture による boundary test。
- README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` の contract 説明。
- root workspace、parser、core、cli、MCP server metadata の version lockstep。

対象外:

- 既存 tool 名、input schema、主要 payload field の破壊的変更。
- `{ metadata, source, payload, draft }` のような大規模再構造化。
- DB への接続、SQL 実行、A5:SQL 設定ファイルや `.a5er` への書き込み。
- prompt injection 文の削除、検閲、変換。

## Design

既存 JSON 形は維持し、top-level に分類 field を追加する。

- `trustedMetadataFields`: trusted guidance として扱える top-level field 名。主に `code`、`message`、`warnings`、`nextAction` と、生成補助 disclosure の固定 field。
- `sourceMetadataFields`: 取得元や parser に関する top-level field 名。主に `filePath`、`kind`、`encoding`、`fileEncoding`、`asset`、`parser`。
- `untrustedPayloadFields`: A5:SQL 由来またはそれを元にした読み取り payload の top-level field 名。主に `text`、`content`、`tables`、`table`、`columns`、`relationships`、`statements`、`assets`、`summary`、`issues`、`suggestions`。
- `draftOutputFields`: 生成補助 draft の top-level field 名。主に `sql`、`mermaid`、`markdown`、`files`、`plan`、`operations`。
- `draftIsDerivedFromUntrustedInput: true`: 生成補助 output が A5:SQL 由来 input から作られた案であることを示す。

分類 field は「存在する field の分類」を示す。存在しない field は list に入れない。これにより client は、既存 payload を壊さずに top-level field を扱える。

実装は `packages/cli/src/mcp/output-contract.ts` に集約する。既存の `contentIsUntrusted: true` は維持し、read-only output は `withUntrustedPayloadContract`、生成補助 output は `withDraftOutputContract` で包む。

## Testing

TDD で次を固定する。

- prompt injection 文を含む `.a5er` helper output は、`trustedMetadataFields` と `untrustedPayloadFields` を返す。
- prompt injection 文は payload field には残るが、`nextAction`、`message`、`code`、`warnings` には混ざらない。
- `search_a5sql_assets` と `parse_a5sql_asset` は asset/snippet/statement payload を `untrustedPayloadFields` に分類する。
- 生成補助 tool は `outputKind: "draft"` に加え、`draftIsDerivedFromUntrustedInput: true` と `draftOutputFields` を返す。
- version は root workspace、parser、core、cli で `0.9.3` に揃える。

最終検証は `rtk pnpm release:check` と `rtk pnpm published:check` とする。

## Success Criteria

- AI が trusted guidance と読むだけの payload を top-level classification から区別できる。
- A5:SQL 由来の prompt injection 文が `warnings`、`message`、`code`、`nextAction` に混ざらない。
- 生成補助 output が、未信頼 input 由来の review 用 draft であることを機械的に判定できる。
- README / AGENTS.md / skill が同じ contract を説明している。
