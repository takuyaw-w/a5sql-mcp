# a5sql-mcp 0.10.0 Architecture Pattern Classification and Tool Description Audit Design

## 背景

`ROADMAP.md` の 0.10.0 は、1.0.0 の public contract を変えずに、現行 MCP server を architecture pattern の観点で分類し、`tools/list` の tool description と anti-pattern を監査する milestone である。

この repo はローカル A5:SQL 資産を読み取り専用で扱う MCP server であり、主分類は Resource Gateway / Domain-Specific Adapter とする。A5:SQL / A5:ER 固有の root 境界、文字コード、解析、秘密情報マスク、untrusted content の分類を LLM-facing API の手前で吸収する。

`generate_*`、`review_*`、`suggest_*`、`compare_*` は Tool Orchestrator として拡張しない。これらは read-only な analysis / draft helper として扱い、DB 実行、ファイル書き込み、資格情報復号には広げない。

## 目的

- README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` に architecture pattern の位置づけを短く残す。
- `tools/list` の tool description だけで、LLM が「いつ使うか」「何を返すか」「read-only か」「draft output か」を判断しやすくする。
- God Tool、unsanitized resource content、同期的な長時間処理、曖昧な tool description の anti-pattern 観点を repo 内に再利用可能な形で残す。
- 既存の 1.0.0 public contract を壊さない。

## 非目的

- tool 名、input schema、主要 output field の変更。
- tool の追加、削除、profile 化。profile 化は 0.10.1 の範囲とする。
- MCP Resource の追加。Resource Gateway pilot は 0.10.3 の範囲とする。
- DB 接続、SQL 実行、A5:SQL 設定や ER 図ファイルへの書き込み、資格情報の復号・表示。
- `tool-outputs.ts` 周辺の大規模リライト。

## 設計

### Docs

README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` に 0.10.0 の節または段落を追加する。

内容は次の境界に揃える。

- 主分類は Resource Gateway / Domain-Specific Adapter。
- `roots` / `A5SQL_MCP_ROOTS`、assetId、範囲読み取り、limit、paging により、ローカル資産を段階的に読む。
- A5:SQL 由来の本文、コメント、識別子、SQL、draft は untrusted content として扱う。
- `generate_*` は experimental draft tool、`review_*`、`suggest_*`、`compare_*` は stable read-only analysis / comparison tool。
- Stateful Session Server は 1.0.0 前の既定方針にしない。再現性のため、明示的な引数で読む。
- DB 接続、SQL 実行、書き込み、資格情報復号は non-goal のまま。

### Tool Description

`packages/cli/src/mcp/server.ts` の description を互換性ある文言改善に限定して監査する。

各 tool description は、可能な範囲で次を含める。

- いつ使うか。
- 何を返すか。
- DB に接続しない、SQL を実行しない、ファイルシステムに書き込まないなどの read-only 境界。
- draft tool は `experimental draft tool` marker を維持し、review 用の案であることを示す。
- root 指定が必要な asset / connection 系 tool は `roots` または `A5SQL_MCP_ROOTS` の明示を説明する。

description 改善では、日本語表現の詳細は変えすぎず、既存 test と docs の分類に沿う。

### Test Guard

`packages/cli/test/mcp-server-smoke.test.ts` に 0.10.0 の description audit guard を追加する。

guard の方向性:

- 既存の expected tool name、stable read-only / experimental draft 分類を再利用する。
- 全 tool の description が空でないことに加え、最低限の選択材料を持つことを確認する。
- draft tool は `experimental draft tool` を含む。
- stable read-only tool は `experimental draft tool` を含まない。
- DB 非接続や root 境界など、tool の性質ごとに必要な文言が抜けた場合に失敗する。

`packages/cli/test/docs-redaction.test.ts` に docs drift guard を追加する。

guard の方向性:

- public guidance files に `0.10.0` と `Architecture Pattern Classification` 相当の説明があること。
- Resource Gateway、Domain-Specific Adapter、anti-pattern、tool description、DB 非接続 / 書き込みなし / 資格情報復号なしの境界を確認する。

## バージョンと ROADMAP

0.10.0 の実装として、root workspace、parser、core、cli、MCP server metadata、smoke test expectation の version を `0.10.0` に揃える。

実装完了時に `ROADMAP.md` の `0.10.0` を `[x]` に変更する。0.10.1 以降は未完了のまま残す。

## 検証

- `rtk pnpm agent:preflight`
- 0.10.0 で追加・変更した focused tests
- `rtk pnpm release:check`
- `rtk pnpm published:check`

  0.10.0 は release candidate 系の続きなので、local source tree だけでなく installed package 経由の MCP startup と `tools/list` も確認する。

## リスク

- description を改善しすぎると、1.0.0 前の API freeze の意味が薄くなる。文言改善に留め、tool 名、input schema、主要 output field は変えない。
- guard test を厳しくしすぎると、自然な説明文の修正まで brittle になる。固定文言の完全一致ではなく、分類と安全境界に必要な語を中心に確認する。
- architecture pattern を理由に Resource を追加したくなるが、それは 0.10.3 へ分ける。

## 承認事項

この design は option 1 の「docs + description + guard test の最小実装」で進める。0.10.0 では public contract を壊さず、architecture pattern の分類、tool description の明確化、anti-pattern audit の再利用可能化に集中する。
