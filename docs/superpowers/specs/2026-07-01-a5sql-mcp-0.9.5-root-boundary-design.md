# a5sql-mcp 0.9.5 Root Boundary Hardening Design

## Goal

0.9.5 では、`roots` / `A5SQL_MCP_ROOTS` の最小権限境界を、既存実装、テスト、docs、version metadata で一貫させる。新しい探索機能は増やさず、1.0.0 前に root 未指定時の安全側 contract を固定する。

## Scope

対象にするもの:

- `search_a5sql_assets`、`read_a5sql_asset`、`parse_a5sql_asset`、`list_a5sql_connections` の root 必須 contract。
- root 未指定時に OS、home、APPDATA、LOCALAPPDATA、USERPROFILE、Wine などの既定候補を探索しないこと。
- `detect_a5sql_locations` は候補提示だけを行い、見つけた path を自動で asset 探索 root にしないこと。
- 明示 path 読み取りでは `roots` が必須であり、root 外 path と root 配下 symlink から外部 path を読めないこと。
- README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` の root boundary 説明。
- root workspace、parser、core、cli、MCP server metadata の version lockstep。

対象外:

- home や drive 全体を指定した場合の実行時 warning 追加。
- 既存 success response から path metadata を削除する破壊的変更。
- `detect_a5sql_locations` の候補生成ロジックの拡張。
- DB 接続、SQL 実行、A5:SQL 設定ファイルや `.a5er` への書き込み。

## Design

既存の root 解決方針を維持する。asset 系 tool は、tool input の `roots` がある場合はその root だけを使い、ない場合は `A5SQL_MCP_ROOTS` だけを使う。どちらもない場合、platform default や home 由来の候補は探索対象にしない。

MCP handler は root が必要な tool で `roots_required` を返し、`nextAction` は `detect_a5sql_locations` で候補を確認して必要最小限の root を明示する固定 guidance にする。`detect_a5sql_locations` の結果はあくまで候補であり、自動的に `search_a5sql_assets`、`read_a5sql_asset`、`parse_a5sql_asset`、`list_a5sql_connections` の許可 root にはならない。

明示 path 読み取りは、`roots` を必須とする。`read_a5sql_asset` は realpath ベースで対象 file と root を比較し、root 外 path と root 内 symlink から外部 file への読み取りを拒否する。拒否時の public output は、対象 path、root、file content を含めない。

0.9.5 は挙動追加よりも contract 固定を優先する。既存テストで十分にカバーされている項目は維持し、不足が見つかった場合だけ TDD でテストを追加し、必要最小限の実装修正を行う。

## Testing

TDD で次を固定または確認する。

- core の asset search/read は、`roots` と `A5SQL_MCP_ROOTS` がない場合に platform/home default を探索しない。
- MCP handler は `read_a5sql_asset`、`search_a5sql_assets`、`parse_a5sql_asset`、`list_a5sql_connections` の root 未指定時に `roots_required` を返す。
- 明示 path 読み取りは `roots` なしで拒否され、root 外 path と root 内 symlink から外部 content を読まない。
- `detect_a5sql_locations` は候補を返すだけで、候補 path を後続 asset tool の root として暗黙利用しないことを docs と test で確認できる。
- README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` は、広すぎる root を避ける説明、目的別 root 例、候補提示と探索 root の違いを同じ contract として説明する。
- version は root workspace、parser、core、cli、MCP server metadata で `0.9.5` に揃える。

最終検証は `rtk pnpm release:check` と `rtk pnpm published:check` とする。

## Success Criteria

- root 制限の挙動が docs と test で一致している。
- 利用者が home や drive 全体を指定する前に、A5:SQL 設定ディレクトリ、保存済み SQL ディレクトリ、対象プロジェクトの作業ディレクトリなど、より狭い root を選びやすい説明になっている。
- root 未指定、root 外 path、symlink 経由の外部 path で、local content、absolute path、secret value が public output に混ざらない。
- package と MCP server metadata が 0.9.5 として揃っている。
