# A5SQL MCP 0.6 Parser Design

## Goal

0.6.0 では、A5:SQL Mk-2 の実運用ファイルに対する読み取り成功率を上げる。主な対象は `.a5er` parser、SQL statement splitter、文字コード検証、parser status / warning の説明である。DB 接続、SQL 実行、A5:SQL 設定ファイルや ER 図ファイルへの書き込みは扱わない。

## Scope

対象は parser と、その結果を core / CLI / docs へ流す契約に限る。

- `.a5er` variant fixture を追加し、headerless、View、Index、Position、PageInfo、DomainInfo、CommonField、未知/欠損セクションの扱いを確認する。
- UTF-8、Shift_JIS、UTF-16LE の読み取り結果と、A5:ER ヘッダーの `ENCODING` 差分 warning をテストで固める。
- SQL splitter の quote / comment 処理を改善し、comment や quoted body 内の semicolon で statement を誤分割しない。
- `parseStatus` と warnings の説明を README に追加し、文字化けや unknown 時の次アクションを追いやすくする。
- workspace、parser、core、cli、MCP server metadata を 0.6.0 に揃える。

## Out of Scope

0.6.0 では次を扱わない。

- 接続先 DB への SQL 実行。
- A5:SQL 設定ファイルや `.a5er` ファイルへの書き込み。
- 実ユーザー環境の A5:SQL ファイルの取り込み。
- parser の大規模分割。
- `packages/cli/src/mcp/tool-outputs.ts` の責務分割。
- 新しい ORM、migration framework、Web UI、daemon。

## Architecture

既存の 3 層構成を維持する。

- `packages/parser`: A5ER parser と SQL splitter の純粋関数を強化する。ファイル I/O や MCP 固有の事情は持ち込まない。
- `packages/core`: ファイル読み取り後の encoding と parser 呼び出しの契約をテストで固める。`parseA5sqlAsset` や単一ファイル parse 経路で warnings が欠落しないことを確認する。
- `packages/cli`: MCP server version metadata と README に関わる範囲だけを更新する。0.6.0 では MCP tool の新規追加はしない。

A5ER parser は現状の `packages/parser/src/index.ts` を維持し、必要な helper は同ファイル内に小さく追加する。SQL splitter は `packages/parser/src/sql.ts` の state machine を拡張し、line comment、block comment、PostgreSQL dollar quote を扱えるようにする。

## Data Flow

A5ER の流れは次のまま維持する。

1. core がファイル本文を読み取り、実デコード encoding を把握する。
2. parser の `parseA5erIni(text, { fileEncoding })` が、ヘッダー上の `ENCODING` と実デコード encoding を比較する。
3. parser は A5ER らしいヘッダーまたは主要セクションがなければ `parseStatus: "unrecognized"` と `a5er_structure_not_recognized` を返す。
4. core / CLI はこの status と warnings を消さずに MCP の `structuredContent` まで流す。

SQL の流れでは、`parseSqlStatements` が最大 100 statements の既存上限を維持し、分割単位だけを改善する。single quote、double quote、backtick、line comment、block comment、dollar quoted body 内の semicolon は delimiter と見なさない。referenced table 抽出は既存の `from` / `join` / `into` / `update` ベースを維持する。

## Parser Behavior

A5ER parser は、認識できる情報だけを安全に構造化する。

- `View` は `objectType: "view"` として返す。
- `Index` は名前、unique 判定、対象カラムを返す。
- `Position` は page、座標、サイズを返す。
- `PageInfo`、`DomainInfo`、`CommonField` は manager 配下の配列として返す。
- 未知セクションは無視する。
- 主要セクションや A5ER ヘッダーがない文書全体は `unrecognized` とする。

存在しない optional 情報は warning にしない。欠損を warning にするのは、認識済み A5ER で manager section がない場合や、table / relationship として成立しない section など、次に確認すべき状態に限る。

## Error Handling

エラーや曖昧な入力は安全側に倒す。

- 未知ファイルは空の正常スキーマとして扱わず、`parseStatus: "unrecognized"` と `a5er_structure_not_recognized` を返す。
- ヘッダーの `ENCODING` と core が渡す `fileEncoding` が違う場合は、既存の `a5er_encoding_mismatch:<declared>:<decoded>` 形式を維持する。
- SQL splitter は不完全な quote / comment / dollar quote があっても throw せず、残りを 1 statement として返す。
- parser は SQL を補正しない。読み取り専用の解析結果として statement preview と参照テーブル候補を返す。
- warnings に実ファイルパス、秘密値、長い本文抜粋を入れない。

## Testing

テストは TDD で追加する。

- `packages/parser/test/a5er.test.ts`: `.a5er` variant fixture を追加し、View、Index、Position、PageInfo、DomainInfo、CommonField、unknown section、欠損 section を確認する。
- `packages/parser/test/sql.test.ts`: line comment、block comment、dollar quote、escaped / doubled quote の semicolon ケースを追加する。
- `packages/core/test/parsers.test.ts`: UTF-8、Shift_JIS、UTF-16LE の decoded content と warning propagation を確認する。
- `packages/cli/test/mcp-server-smoke.test.ts`: server metadata と package version を 0.6.0 に更新する。
- `README.md`: `parseStatus`、`a5er_structure_not_recognized`、`a5er_encoding_mismatch`、文字化け時の次アクションを説明する。
- 最後に `rtk pnpm release:check` を実行し、format、lint、build、test、typecheck、pack dry-run を確認する。
- pack dry-run 後に root `.tgz` が残っていないことを確認する。

## Versioning

0.6.0 として root workspace、parser、core、cli、`A5SQL_MCP_SERVER_VERSION` を lockstep で揃える。release workflow は package versions と tag の整合性を見るため、関連 version は個別にずらさない。

## Compatibility

既存の public API、MCP tool 名、入力 schema は変更しない。parser の返却 shape は既存フィールドを維持し、改善は fixture 増加、SQL split 精度、warnings の伝播確認、docs の明確化に留める。
