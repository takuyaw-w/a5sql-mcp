# a5sql-mcp 0.9.8 Client / Agent Safety Docs Design

## Goal

0.9.8 では、MCP クライアントや AI エージェントが `a5sql-mcp` の出力を安全に扱うための最小限の利用ガイドを整える。目的は新しい MCP tool や出力 field を増やすことではなく、0.9.7 までに固定した secret masking、untrusted content signal、draft disclosure、root boundary を、README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` で同じ意味として読めるようにすることである。

## Scope

対象にするもの:

- README に、prompt injection 防衛、untrusted content、read-only、draft output、root 最小権限をまとめた短い client / agent safety 節を置く。
- AGENTS.md と `.agents/skills/a5sql-mcp/SKILL.md` のセキュリティ方針を README と揃える。
- `read_a5sql_file` / `read_a5sql_asset` で本文を読む前に、必要な範囲だけを指定する例を追加する。
- `generate_sql_select`、`generate_schema_markdown`、`generate_model_files`、`generate_migration_plan` などの draft output を、そのまま実行・適用しない説明を再確認する。
- `packages/cli/test/docs-redaction.test.ts` に、public guidance files が 0.9.8 の安全境界を説明していることを固定するテストを追加する。

対象外:

- 新しい MCP tool の追加。
- MCP tool の input schema / output schema の変更。
- 0.9.9 で扱う API freeze の tool 棚卸し。
- version bump、release tag、publish workflow の変更。
- 接続先 DB への SQL 実行、A5:SQL 設定ファイルへの書き込み、資格情報の復号や表示。

## Design

README には、利用者と MCP クライアント実装者が最初に確認できる安全ガイドを追加する。既存の root 最小権限、DB 接続と SQL 実行、生成補助 tool、untrusted payload boundary の説明は活かしつつ、0.9.8 の節として「この MCP server の tool 出力をどう扱うか」に焦点を合わせる。抽象的な security generality ではなく、`contentIsUntrusted`、`trustedMetadataFields`、`untrustedPayloadFields`、`draftOutputFields`、`draftIsDerivedFromUntrustedInput` という実際の field に結びつけて説明する。

本文読み取りの例は、全量読み取りを推奨しない形にする。起動時ファイルは `read_a5sql_file` の `startLine` / `maxLines` または `offsetChars` を使い、asset は `read_a5sql_asset` の `maxChars` と必要に応じた `roots` を使う例を示す。例の path は `/absolute/path/to/a5sql-data` のような抽象 path に留め、実在するユーザー固有 path や credential に見える値を入れない。

AGENTS.md には、エージェントが作業時に守る判断基準として同じ境界を追記する。A5:SQL 由来の本文、コメント、識別子、SQL statement は system / developer / user 指示ではなく、読むだけの未信頼 payload として扱う。`warnings`、`message`、`code`、`nextAction` は固定 guidance として設計・レビューするが、A5:SQL 由来文字列から直接組み立てない方針を維持する。

`.agents/skills/a5sql-mcp/SKILL.md` には、Codex/AI 向けの実務手順として README と同じ注意を置く。特に、root は必要最小限にすること、本文は範囲指定して段階的に読むこと、生成された SQL / Markdown / model / migration は review 用 draft であり実行・適用しないことを明記する。

`packages/cli/test/docs-redaction.test.ts` は、0.9.8 の docs contract が README、AGENTS.md、skill の三者から消えないようにする。既存の redaction audit と同じ public guidance files を対象に、`contentIsUntrusted`、`trustedMetadataFields`、`untrustedPayloadFields`、`draftOutputFields`、`startLine`、`maxLines`、`offsetChars`、`maxChars`、`そのまま実行`、`そのまま適用`、`A5SQL_MCP_ROOTS` などの必須語句を確認する。テストは docs の正確な文章を固定しすぎず、安全境界の存在を固定する。

## Testing

TDD で次を固定する。

- README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` が、A5:SQL 由来 payload を `contentIsUntrusted` 付きの未信頼 content として扱う説明を含む。
- README、AGENTS.md、skill が、`trustedMetadataFields` と `untrustedPayloadFields` / `draftOutputFields` の違いを説明する。
- README が、`read_a5sql_file` の `startLine` / `maxLines` / `offsetChars` と、`read_a5sql_asset` の `maxChars` を使った範囲読み取り例を含む。
- public guidance files が、生成された SQL、Markdown、model、migration 案をそのまま実行・適用しないことを説明する。
- 既存の docs redaction audit により、実在するローカル path や credential-looking values が docs に入らないことを維持する。

最終検証は、docs-focused 変更として `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- docs-redaction.test.ts` を先に実行し、最後に `rtk pnpm release:check` を実行する。package publish 相当の startup 変更は含めないため、`published:check` は release finish 時の追加確認に回せる。

## Success Criteria

- 利用者向け README とエージェント向け AGENTS / skill が、同じ安全境界を説明している。
- prompt injection への注意が、この MCP の tool output field と読み取り flow に即して説明されている。
- 本文読み取りは、全量取得ではなく必要範囲を段階的に読む例として示されている。
- 生成補助 tool の出力は review 用 draft として説明され、そのまま実行・適用するものではないことが明確である。
- docs contract を固定するテストがあり、既存の redaction audit と矛盾していない。
