# a5sql-mcp 0.9.6 Parser Robustness for Hostile / Weird Files Design

## Goal

0.9.6 では、壊れたファイル、未知の A5:ER variant、文字コード不一致、巨大ファイル、prompt injection を含む本文を扱っても、parser / core / CLI / MCP output が安全側に倒れる contract を固定する。主目的は parser の大規模刷新ではなく、unparseable な入力を正常な空 schema と誤認せず、失敗理由と次の確認手順を安定して返すことである。

## Scope

対象にするもの:

- synthetic fixture による、未知 `.a5er`、途中で切れた `.a5er`、文字コード不一致 `.a5er`、壊れた SQL、巨大 text asset、prompt injection 風文字列を含む payload の確認。
- `a5er_structure_not_recognized` と `a5er_encoding_mismatch:<declared>:<decoded>` の warning / `parseStatus` / `nextAction` contract。
- SQL splitter が quote、comment、PostgreSQL dollar quote、壊れた quote を含む入力で crash せず、statement 境界を過度に誤分割しないこと。
- secret masking、output boundary、untrusted payload 境界と parser warning / message / nextAction の責務分離。
- README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` の parser robustness 説明。
- root workspace、parser、core、cli、MCP server metadata の version lockstep。

対象外:

- A5:ER parser の全面的な再設計、形式推定 engine の追加、または未知 variant の深い復元。
- SQL 実行、DB 接続、資格情報の復号、A5:SQL 設定や `.a5er` への書き込み。
- 0.9.7 で扱う MCP adversarial end-to-end session replay の前倒し。
- 既存 output schema の破壊的変更。
- 巨大ファイルを全量 parse するための制限緩和。

## Design

既存 parser の責務分離を維持する。A5:ER parser は、認識できる header または section 構造がない入力を `parseStatus: "unrecognized"` とし、正常な空 schema として扱わない。途中で切れた `.a5er` や未知 section を含む `.a5er` も、table / relationship として復元できる情報だけを bounded に扱い、必要な構造が欠ける場合は warning と `nextAction` で確認手順を返す。

文字コード不一致は、A5:ER header 上の宣言と実際に decode された encoding を比較して warning に残す。warning code は固定文字列だけで構成し、A5:SQL 由来の本文やコメントを混ぜない。利用者が確認すべき手順は `read_a5sql_file` または該当 asset 読み取りで先頭行、文字コード、ファイル形式を確認する guidance に寄せる。

SQL splitter は、quote、line comment、block comment、backtick quote、dollar quote 内の semicolon を statement 区切りとして扱わない。壊れた quote や途中で切れた comment があっても parser は crash せず、返す statement 数、snippet、warnings は既存上限と secret masking の範囲に収める。

巨大 text / asset は、初期読み取り上限、文字数上限、`truncated`、`hasMore`、`nextAction` で段階読み取りに誘導する。0.9.6 では全量 parse を試みる条件を広げず、file too large / truncated の contract を test と docs で確認する。

Prompt injection 風の文字列は untrusted payload としてだけ扱う。table / column comment、SQL comment、本文 excerpt など A5:SQL 由来の値が `warnings`、`message`、`code`、`nextAction` に混ざらないことを確認する。必要に応じて `contentIsUntrusted`、`trustedMetadataFields`、`sourceMetadataFields`、`untrustedPayloadFields`、`draftOutputFields` の説明を docs 側で補強する。

実装修正は、TDD で contract gap が見つかった箇所に限定する。候補は、unrecognized `.a5er` の helper output、`parse_a5sql_asset` の parser metadata、SQL splitter の壊れた入力 handling、巨大 asset の `nextAction` 表示である。既存 contract を満たしている箇所は、fixture と regression test だけを追加して固定する。

## Testing

TDD で次を固定または確認する。

- parser は未知 `.a5er` と途中で切れた `.a5er` を正常な空 schema と誤認せず、`parseStatus` と warning を返す。
- parser は文字コード宣言と decode 結果がずれた `.a5er` で `a5er_encoding_mismatch:<declared>:<decoded>` を返し、本文由来の文字列を warning code に混ぜない。
- parser は malformed entity / relationship / unknown section を、復元可能な範囲で bounded に扱い、欠損情報を warning として返す。
- SQL parser は comment / quote / dollar quote 内 semicolon を分割せず、壊れた quote や comment を含む入力でも crash しない。
- core の asset parse / read は、巨大 text と巨大 `.a5er` を上限内で扱い、`truncated`、`hasMore`、`file_too_large`、`nextAction` の contract を維持する。
- MCP tool output は unrecognized `.a5er` と encoding mismatch で、利用者が次に確認すべき tool と観点を返す。
- Hostile payload は output の untrusted field には現れてよいが、trusted guidance field には混ざらない。
- README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` は、壊れたファイルの扱い、`parseStatus` の確認、prompt injection payload の境界を同じ contract として説明する。
- version は root workspace、parser、core、cli、MCP server metadata で `0.9.6` に揃える。

最終検証は `rtk pnpm release:check` と `rtk pnpm published:check` とする。

## Success Criteria

- unparseable な `.a5er`、壊れた SQL、巨大 text、hostile content が crash や正常 schema 誤認につながらない。
- 失敗時 output は、固定 code / warning / message / nextAction で次の確認手順を説明し、A5:SQL 由来の untrusted payload を trusted guidance に混ぜない。
- 既存の secret masking、root boundary、output boundary contract と矛盾しない。
- parser / core / CLI / MCP の relevant tests と release readiness check が通る。
- package と MCP server metadata が 0.9.6 として揃っている。
