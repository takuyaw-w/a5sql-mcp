# A5SQL MCP 0.5 Contract Design

## Goal

0.5.0 では、1.0.0 で安定 API として扱う MCP tool の JSON 出力契約を揃える。DB 接続、設定ファイル書き込み、資格情報の復号や表示は扱わない。

## Scope

対象は MCP の read-only 公開面に限る。

- 主要 tool の `structuredContent` に、呼び出し側が次の行動を判断しやすいメタデータを揃える。
- `search_a5sql_assets` に effective limit と探索打ち切り理由を追加する。
- stdio MCP server を in-memory client から起動し、`tools/list` と代表 tool の `structuredContent` を検証する。
- README に安定 tool と生成補助 tool の位置づけを書く。

## Out of Scope

0.5.0 では `tool-outputs.ts` の大規模分割は行わない。これは 0.8.0 の作業に残す。

## Architecture

既存の責務分離を維持する。

- `packages/core`: asset 探索の事実を返す層。従来の `searchA5sqlAssets()` は互換維持のため配列を返し続け、追加 API で探索メタデータを返す。
- `packages/cli/src/mcp/tool-handlers.ts`: MCP tool の `structuredContent` を整形する層。core の探索メタデータを MCP レスポンスへ載せる。
- `packages/cli/src/mcp/server.ts`: tool 登録と MCP server metadata を持つ層。登録構造は大きく変えない。
- `README.md`: 安定 read-only tool と生成補助 tool の位置づけを説明する。

## Data Flow

`search_a5sql_assets` の流れは次のとおり。

1. MCP client が `search_a5sql_assets` を呼ぶ。
2. CLI handler が入力値を受け取る。
3. handler が core の metadata 付き探索 API を呼ぶ。
4. core が root 配下を読み取り専用で探索し、asset 一覧と探索メタデータを返す。
5. handler が MCP 向け `structuredContent` を組み立てる。

既存フィールドは削除しない。`query`, `roots`, `count`, `truncated`, `nextAction`, `assets` は残し、`effectiveLimit`, `returnedAssetCount`, `visitedFileCount`, `cutoffReason`, `warnings` を追加する。

## Contract Behavior

出力契約では次のフィールドを必要な tool に安定して含める。

- `found`: 単一対象の検索や読み取りで、対象が見つかったかを表す。
- `truncated`: 内容または一覧が上限で切り詰められたかを表す。
- `hasMore`: ページングで続きがあるかを表す。
- `warnings`: 呼び出し成功時でも注意が必要な状態を機械的に伝える。
- `nextAction`: ユーザーまたは AI が次に確認すべきことを日本語で短く示す。

`search_a5sql_assets` は既存互換を保ったまま、次を追加する。

- `effectiveLimit`: 実際に適用された返却上限。
- `cutoffReason`: `null`、`limit_exceeded`、`max_files_reached` などの機械可読な打ち切り理由。

## Error Handling

エラーや上限到達時の扱いは既存方針を維持する。

- 対象が見つからない単一取得系は `found: false` と `code` / `message` / `nextAction` を返す。
- 一覧系やページング系は、続きがある場合に `truncated` または `hasMore` を返す。
- `warnings` は成功レスポンスでも注意が必要な状態を伝えるために使う。
- `nextAction` は人間と AI が次に取るべき行動を短く示す。
- `search_a5sql_assets` は `cutoffReason` を返し、打ち切り理由を機械的に追えるようにする。
- DB 接続、SQL 実行、ファイル書き込み、資格情報の復号・表示は行わない。

## Tool Groups

README では tool を次のように分けて説明する。

- 安定 read-only tool: ローカル A5:SQL 資産の検出、検索、読み取り、解析、テーブル/カラム/リレーション探索。
- 生成補助 tool: SQL、Mermaid、Markdown、model、migration plan、review/suggest 系。いずれも案を返すだけで、ファイルシステムや DB には書き込まない。

## Testing

テストは TDD で追加する。

- `packages/core/test/assets.test.ts`: metadata 付き asset 探索 API の `max_files_reached` を確認する。
- handler 直接テストで `search_a5sql_assets` の `effectiveLimit` と `cutoffReason` を確認する。
- MCP smoke test で `tools/list` に代表 tool が存在することと、代表 `callTool` の `structuredContent` が契約フィールドを持つことを確認する。
- 既存の asset/security test が秘密情報を返さないことを引き続き確認する。
- 最後に `rtk pnpm release:check` で format、lint、build、test、typecheck、pack dry-run を確認する。

## Versioning

0.5.0 として root workspace、parser、core、cli、`A5SQL_MCP_SERVER_VERSION` を lockstep で揃える。

## Compatibility

既存フィールドは削除しない。新しいフィールドは追加に留める。レスポンス本文のマスク方針、root 制限、DB 非接続の境界は変更しない。
