# a5sql-mcp 0.9.4 Redaction and Path Privacy Audit Design

## Goal

0.9.4 では、秘密情報とユーザー固有 path が、代表的な MCP tool 出力、error、warning、docs、fixture に残らないことを再監査し、1.0.0 前の安全境界として固定する。

## Scope

対象にするもの:

- password、token、API key、private key、URL userinfo、ODBC connection string、JSON/env 形式の credential。
- `read_a5sql_file`、`read_a5sql_asset`、`search_a5sql_assets`、`parse_a5sql_asset`、`list_a5sql_connections` の代表出力。
- roots 未指定、asset not found、root 外 path などの error output。
- README / docs / AGENTS / skill に、実在 path や credential に見える値がないこと。
- root workspace、parser、core、cli、MCP server metadata の version lockstep。

対象外:

- 新しい secret scanner の導入。
- path metadata を返す既存成功 contract の破壊的変更。
- DB 接続、SQL 実行、資格情報の復号・表示。
- 0.9.5 で扱う root boundary hardening の追加仕様。

## Design

既存の `maskSensitiveText` と MCP handler の public output shape は維持する。0.9.4 では、不足している代表ケースを TDD で固定し、漏れが見つかった箇所だけを最小修正する。

主な確認点:

- raw text / SQL / JSON / env / ODBC / URL userinfo / PEM private key は、read、search snippet、parse result に raw value として残らない。
- `list_a5sql_connections` は connection candidate の存在と非秘密 field だけを返す。`revealNonSecret: true` でも password、token、完全な DSN / URL / ODBC connection string は返さない。
- error、warning、`nextAction` は固定 guidance のみで構成し、ユーザー固有 root、root 外 path、secret value、file content を含めない。
- docs の例は `/absolute/path/...` のような placeholder に留め、実在 path や本物に見える credential を置かない。

connection 抽出では、`database_url`、`dsn`、`connection_string` などの key は secret-like として扱い、field value として public output に採用しない。これにより、`revealNonSecret: true` の場合でも完全な接続文字列を返さない。

## Testing

TDD で次を固定する。

- core の `maskSensitiveText` が JSON/env/ODBC/URL userinfo/PEM private key をまとめて伏せる。
- core の `extractConnectionCandidate` が `DATABASE_URL` や `ODBC_CONNECTION_STRING` を public field にしない。
- MCP asset tool の read/search/parse/list connection output を JSON stringify しても raw credential が残らない。
- root 未指定、asset not found、root 外 path の error output に secret content、root、absolute path が残らない。
- docs audit test で、README / docs / AGENTS / skill に forbidden local path や credential marker がないことを確認する。
- version は root workspace、parser、core、cli で `0.9.4` に揃える。

最終検証は `rtk pnpm release:check` と `rtk pnpm published:check` とする。

## Success Criteria

- 代表的な credential 形式が public output に残らない。
- connection candidate は存在確認に必要な情報だけを返し、秘密値や完全な接続文字列を返さない。
- error / warning / nextAction に、ユーザー固有 path や secret value が混ざらない。
- README / docs / AGENTS / skill が placeholder path と安全な説明だけを含む。
