# a5sql-mcp 0.4 Design

## 目的

0.4.0 では、`packages/core` に既にある A5:SQL 資産探索・読み取り・接続候補抽出を MCP の公開 tool として使えるようにする。`search_a5sql_assets` で見つけた資産を次に安全に読む流れを作り、A5:SQL の設定場所や接続候補を、秘密情報を返さない形で確認できるようにする。

このリリースでは、接続先 DB へのアクセス、SQL 実行、A5:SQL 設定ファイルへの書き込み、資格情報の復号・表示は扱わない。

## スコープ

追加する MCP tool:

- `detect_a5sql_locations`
  - A5:SQL の設定ディレクトリ候補を返す。
  - 候補の存在有無、読み取り可否、検出理由、由来を返す。
  - `A5SQL_MCP_ROOTS` と明示指定 root を扱える既存 core 実装を利用する。
- `read_a5sql_asset`
  - `search_a5sql_assets` の結果から参照できる資産本文を、サイズ制限と秘密情報マスクつきで返す。
  - `.a5er`、`.sql`、text 系資産を対象にし、バイナリや unsupported file は本文を返さず警告を返す。
  - 大きなファイルは `truncated`、返却文字数、制限値を返す。
- `list_a5sql_connections`
  - A5:SQL 設定 root 配下から接続候補を抽出して返す。
  - パスワード、トークン、接続文字列などの秘密情報は常にマスクする。
  - 非秘密値もデフォルトでは安全側に倒し、明示オプションがある場合だけ返す。

更新するドキュメント:

- README に root 指定、asset search から read/parse への流れ、connection 情報のマスク方針を書く。
- AGENTS.md に 0.4.0 で追加される tool を実装済み公開面として反映し、今後の候補から外す。

追加する検証:

- MCP handler 直接テストで、3 tool の登録と主要レスポンスを確認する。
- 可能な範囲で stdio MCP server の E2E smoke test を追加し、`tools/list` と代表 `callTool` を確認する。
- 最終確認で `pnpm release:check` を実行する。

## 非スコープ

- DB へ接続して live schema を読む機能。
- SQL を実行する機能。
- A5:SQL の設定ファイルを書き換える機能。
- 暗号化済みまたは独自形式の資格情報を復号して返す機能。
- 接続候補の値を完全な接続文字列として返す機能。
- 生成系 tool の整理や `tool-outputs.ts` の分割。これは後続の 0.8.0 で扱う。

## アーキテクチャ

既存の責務分離を維持する。

- `packages/core/src/locations.ts`
  - `detectA5sqlLocations` を MCP handler から呼び出す。
  - OS や環境変数ごとの候補生成は core に閉じ込めたままにする。
- `packages/core/src/assets.ts`
  - `readA5sqlAsset` を MCP handler から呼び出す。
  - ファイル分類、文字列読み取り、秘密情報マスク、truncation は core に閉じ込めたままにする。
- `packages/core/src/connections.ts`
  - `listA5sqlConnections` を MCP handler から呼び出す。
  - connection candidate の抽出とマスク判断は core に閉じ込めたままにする。
- `packages/cli/src/mcp/tool-schemas.ts`
  - 3 tool の input schema を追加する。
- `packages/cli/src/mcp/tool-handlers.ts`
  - MCP tool 名から core 関数を呼び出す薄い handler を追加する。
- `packages/cli/src/mcp/server.ts`
  - `server.tool(...)` 登録を追加する。

MCP 層では、既存 core の出力を大きく変換しない。0.4.0 の目的は公開面の穴を埋めることであり、出力契約の全面整理は 0.5.0 に残す。

## 入力と出力

### detect_a5sql_locations

入力:

- `roots?: string[]`
  - 追加で確認したい root 候補。
- `includeDefaults?: boolean`
  - OS や環境変数由来の候補を含めるか。

出力:

- `candidates`
  - 候補 path、source、reason、exists、readable、warnings を含む。
- `totalCandidateCount`
- `returnedCandidateCount`
- `warnings`
- `nextAction`

### read_a5sql_asset

入力:

- `assetId?: string`
  - `search_a5sql_assets` の結果 ID。
- `path?: string`
  - 明示 path。`assetId` とどちらか一方を指定する。
- `roots?: string[]`
  - 読み取り対象を制限する root。
- `maxChars?: number`
  - 返却する最大文字数。
- `offsetChars?: number`
  - 文字単位の開始位置。

出力:

- `found`
- `asset`
  - ID、path、kind、size、modifiedAt を含む。
- `content`
  - マスク済み本文。unsupported や binary の場合は返さない。
- `truncated`
- `offsetChars`
- `returnedChars`
- `maxChars`
- `warnings`
- `nextAction`

### list_a5sql_connections

入力:

- `roots?: string[]`
  - 探索する A5:SQL 設定 root。
- `limit?: number`
  - 返却件数上限。
- `revealNonSecret?: boolean`
  - 非秘密項目を返すかどうか。デフォルトは `false`。

出力:

- `connections`
  - 接続候補 ID、source、kind、masked fields、warnings を含む。
- `totalConnectionCount`
- `returnedConnectionCount`
- `truncated`
- `warnings`
- `nextAction`

## エラー処理

- root が存在しない場合は例外ではなく候補や warning として返す。
- asset が見つからない場合は `found: false` と `nextAction` を返す。
- `assetId` と `path` が両方指定された場合、入力エラーとして扱う。
- `assetId` と `path` がどちらも指定されない場合、入力エラーとして扱う。
- バイナリ、暗号化済み、unsupported file は本文を返さず、形式を確認するための warning と `nextAction` を返す。
- エラー message には秘密情報や長い本文を含めない。

## セキュリティ

- すべて読み取り専用とする。
- DB 接続や SQL 実行は行わない。
- password、token、secret、private key、connection string に相当する値は返さない。
- `read_a5sql_asset` の本文は `maskSensitiveText` または core 側の同等処理を通したものだけ返す。
- `list_a5sql_connections` は秘密値を常にマスクし、非秘密値もデフォルトでは返さない。
- テスト fixture には実在する接続情報やユーザー固有パスを入れない。

## テスト方針

TDD で進める。

1. `packages/cli/test/mcp-asset-tools.test.ts` または新規 MCP test に、未登録 tool が原因で落ちる test を追加する。
2. `detect_a5sql_locations` の tool 登録と handler 応答を確認する。
3. `read_a5sql_asset` が検索結果 ID または path からマスク済み本文を返すことを確認する。
4. `read_a5sql_asset` が unsupported/binary file で本文を返さず warning を返すことを確認する。
5. `list_a5sql_connections` が秘密情報をマスクし、`revealNonSecret` のデフォルトが安全側であることを確認する。
6. stdio MCP server の smoke test を追加できる場合は、`tools/list` に 0.4.0 tool が含まれることと、代表 tool の `callTool` が `structuredContent` を返すことを確認する。
7. 最後に `pnpm release:check` を実行する。

## ドキュメント方針

README では、利用者が次の順に進められるように書く。

1. `detect_a5sql_locations` で候補 root を確認する。
2. `search_a5sql_assets` で root 配下の SQL、ER 図、text を探す。
3. `read_a5sql_asset` で本文を安全に確認する。
4. `.a5er` や `.sql` は必要に応じて `parse_a5sql_asset` に渡す。
5. `list_a5sql_connections` は接続候補の存在確認用であり、資格情報を返さない。

AGENTS.md では、0.4.0 tool を「現時点で公開している tool」に移し、同じ項目を「今後の拡張候補」に残さない。

## 完了条件

- `detect_a5sql_locations`、`read_a5sql_asset`、`list_a5sql_connections` が MCP tool として登録されている。
- 3 tool の入力 schema と handler test がある。
- `search_a5sql_assets` から `read_a5sql_asset` / `parse_a5sql_asset` へ進む流れが README で説明されている。
- connection 情報はデフォルトで秘密値も非秘密値も安全側に倒れている。
- MCP E2E smoke test の土台がある。
- `pnpm release:check` が通る。
