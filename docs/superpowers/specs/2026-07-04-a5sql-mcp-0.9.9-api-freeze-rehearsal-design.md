# a5sql-mcp 0.9.9 API Freeze Rehearsal Design

## Goal

0.9.9 では、1.0.0 で安定扱いにする MCP tool と experimental draft tool の境界を凍結する。新機能追加ではなく、server registration、docs、test の公開面を照合し、1.0.0 前に変えるべき API 名、field 名、tool 分類が残っていない状態にする。

## Context

0.9.8 までに Client / Agent Safety Docs と version lockstep は完了している。現在の `main` は root、parser、core、cli、MCP server metadata が `0.9.8` で揃っており、`v0.9.8` tag も作成済みである。

ROADMAP の 0.9.9 は API Freeze Rehearsal であり、tool 名、description、input schema、output contract の棚卸し、README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` / server registration / test の tool 一覧照合、stable read-only tool と experimental draft tool の分類確認を求めている。

現状の README では `review_a5sql_schema`、`suggest_schema_changes`、`compare_a5er_with_live_schema` が「生成補助 tool」側に並んでいる。一方で server registration ではこれらに `experimental draft tool` description は付いておらず、draft disclosure も返さない。0.9.9 では server の現行 contract に合わせ、生成物を返す 5 tool だけを experimental draft tool として固定する。

## Scope

対象にするもの:

- `.a5er` 起動時に公開される MCP tool 名の exact set をテストで固定する。
- 各 tool の `description` と `inputSchema` が `tools/list` に出ることを確認する。
- experimental draft tool の description に `experimental draft tool` が含まれ、stable read-only tool には含まれないことを確認する。
- README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` の tool 一覧と分類を server registration に合わせる。
- `review_a5sql_schema`、`suggest_schema_changes`、`compare_a5er_with_live_schema` を stable read-only analysis/comparison tool として docs 上で分類する。
- 代表 draft tool の structuredContent が draft disclosure contract を返すことを固定する。
- root、parser、core、cli、`A5SQL_MCP_SERVER_VERSION`、version assertion test を `0.9.9` に揃える。
- `rtk pnpm release:check` と `rtk pnpm published:check` で検証する。

対象外:

- 新しい MCP tool の追加。
- 既存 tool 名の変更や削除。
- input schema の破壊的変更。
- output field name の破壊的変更。
- handler 直接 test を大きく書き換えること。
- tool registration の大規模一元化 refactor。
- release tag、publish、push。
- 接続先 DB への SQL 実行、A5:SQL 設定ファイルへの書き込み、資格情報の復号や表示。

## Stable and Experimental Classification

Stable read-only tool:

- `describe_a5sql_file`
- `parse_a5sql_file`
- `read_a5sql_file`
- `detect_a5sql_locations`
- `read_a5sql_asset`
- `list_a5sql_connections`
- `search_a5sql_assets`
- `parse_a5sql_asset`
- `list_a5sql_tables`
- `describe_a5sql_table`
- `explain_a5sql_table`
- `list_a5sql_relationships`
- `find_a5sql_tables`
- `find_a5sql_columns`
- `review_a5sql_schema`
- `suggest_schema_changes`
- `compare_a5er_with_live_schema`

Experimental draft tool:

- `generate_sql_select`
- `generate_mermaid_er_diagram`
- `generate_model_files`
- `generate_schema_markdown`
- `generate_migration_plan`

この分類では、schema review、schema change suggestion、live schema comparison は A5:ER と外部 snapshot を読み取り専用で分析する stable tool として扱う。SQL、Mermaid、model、Markdown、migration plan のように利用者がレビューする生成物を返す tool は experimental draft tool として扱う。

## Design

MCP server の `tools/list` を source of truth として、0.9.9 用の freeze rehearsal test を追加または既存 smoke test に追加する。テストは in-memory MCP client 経由で `.a5er` 起動時の tool 一覧を取得し、stable read-only tool と experimental draft tool の exact set を確認する。

各 tool について、`name`、`description`、`inputSchema` が存在することを確認する。experimental draft tool は description に `experimental draft tool` を含む必要があり、stable read-only tool は含んではならない。これにより、README 側の分類と server registration の分類 drift を検出しやすくする。

output contract は既存の adversarial E2E を維持しつつ、代表 draft tool である `generate_sql_select` の structuredContent に `outputKind: "draft"`、`readOnly: true`、`writesToFileSystem: false`、`connectsToDatabase: false`、`executesSql: false`、`draftIsDerivedFromUntrustedInput: true`、`draftOutputFields` が含まれることを確認する。stable tool に draft disclosure を強制追加しない。

README の「公開する MCP tool」は、安定 read-only tool に schema review / suggestion / comparison を含め、生成補助 tool は 5 tool に限定する。AGENTS.md と skill の tool 一覧も同じ分類が読み取れるように短く補足する。

runtime behavior は原則として変更しない。もしテストで分類 drift が見つかった場合は、server description と docs のどちらが 1.0.0 contract として妥当かを優先して判断する。今回の design では server registration の現行 behavior を維持し、docs を合わせる。

## Testing

- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-server-smoke.test.ts -t "freezes the 0.9.9 MCP API surface"` で tool 名、description、input schema、stable / draft 分類を focused に確認する。
- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-server-smoke.test.ts -t "reports 0.9.9 version metadata"` で version lockstep を focused に確認する。
- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- docs-redaction.test.ts` で public guidance files の tool 分類と安全境界を確認する。
- `rtk pnpm release:check` で format、lint、build、test、typecheck、pack dry-run を確認する。
- `rtk pnpm published:check` で packed package install 後の MCP startup、`tools/list`、minimal adversarial assertion を確認する。

## Success Criteria

- `.a5er` 起動時の 22 tool が stable read-only tool 17 件、experimental draft tool 5 件として固定されている。
- experimental draft tool の description と structuredContent が draft contract と一致している。
- stable read-only tool が docs と server registration で同じ分類になっている。
- README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md`、server registration、test の tool 一覧に説明可能なズレがない。
- root、parser、core、cli、MCP server metadata、smoke test expectation が `0.9.9` で一致している。
- `release:check` と `published:check` が通る。
