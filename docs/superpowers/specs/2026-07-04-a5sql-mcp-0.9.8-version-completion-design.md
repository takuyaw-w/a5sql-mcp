# a5sql-mcp 0.9.8 Version Completion Design

## Goal

0.9.8 の Client / Agent Safety Docs 実装を、package version と MCP server metadata の lockstep まで完了させる。既存の 0.9.8 docs contract は維持し、0.9.9 の API freeze rehearsal には踏み込まない。

## Context

0.9.8 の README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md`、`packages/cli/test/docs-redaction.test.ts` はすでに入っている。一方で workspace root、parser、core、cli の package version、`A5SQL_MCP_SERVER_VERSION`、smoke test の version assertion はまだ `0.9.7` のままである。

README の開発者向けリリース確認にも `0.9.7` のリリース候補としての説明が残っているため、0.9.8 completion として古い version 表記を整理する。

## Scope

対象にするもの:

- `package.json` の version を `0.9.8` にする。
- `packages/parser/package.json` の version を `0.9.8` にする。
- `packages/core/package.json` の version を `0.9.8` にする。
- `packages/cli/package.json` の version を `0.9.8` にする。
- `packages/cli/src/mcp/server.ts` の `A5SQL_MCP_SERVER_VERSION` を `0.9.8` にする。
- `packages/cli/test/mcp-server-smoke.test.ts` の version assertion を `0.9.8` にする。
- README の開発者向けリリース確認に残る `0.9.7` 表記を、0.9.8 completion 後でも古くならない表現にする。
- `rtk pnpm release:check` と `rtk pnpm published:check` で検証する。

対象外:

- 0.9.9 API Freeze Rehearsal の tool 棚卸し。
- MCP tool の input schema / output contract 変更。
- 新しい MCP tool の追加。
- release tag、publish、push。
- 接続先 DB への SQL 実行、A5:SQL 設定ファイルへの書き込み、資格情報の復号や表示。

## Design

version は package manifests と MCP server metadata を同時に更新し、smoke test の lockstep assertion で drift を検出する。既存の version assertion test は root、parser、core、cli の package version をまとめて確認しているため、期待値だけを `0.9.8` に揃える。

README の開発者向けリリース確認は、0.9.7 固有の adversarial E2E 説明を維持しつつ、現在の release candidate 手順として読めるようにする。`published:check` が package install 後の MCP startup と minimal adversarial assertion を確認する、という現行 contract は変えない。

runtime behavior は変更しない。0.9.8 safety docs の追加済み内容と docs redaction guard を維持し、今回の差分は version completion と stale wording の整理に限定する。

## Testing

- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-server-smoke.test.ts -t "reports 0.9.8 version metadata"` で version lockstep を focused に確認する。
- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- docs-redaction.test.ts` で 0.9.8 safety docs guard が維持されていることを確認する。
- `rtk pnpm release:check` で format、lint、build、test、typecheck、pack dry-run を確認する。
- `rtk pnpm published:check` で packed package install 後の MCP startup、`tools/list`、minimal adversarial assertion を確認する。

## Success Criteria

- root、parser、core、cli、MCP server metadata、smoke test expectation が `0.9.8` で一致している。
- README の開発者向けリリース確認が 0.9.7 固有の stale wording として読まれない。
- 0.9.8 safety docs contract は維持されている。
- `release:check` と `published:check` が通る。
