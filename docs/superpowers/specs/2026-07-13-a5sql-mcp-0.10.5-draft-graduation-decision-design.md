# a5sql-mcp 0.10.5 Dry Run / Draft Graduation Decision 設計

## 背景

0.10.5 は、0.10.x で追加・監査した architecture、tool profile、contract integrity、platform contract、CI / publish、MCP Resource Gateway を release gate で再確認し、5つの生成補助 tool を 1.0.0 前に stable read-only draft helper へ卒業させるか判断する milestone である。

対象 tool は次の5つとする。

- `generate_sql_select`
- `generate_mermaid_er_diagram`
- `generate_model_files`
- `generate_schema_markdown`
- `generate_migration_plan`

現在の tool description は5 tool を `experimental draft tool` として分類している。卒業後も、生成結果をそのまま実行・適用できるという意味にはしない。`outputKind: "draft"`、review 前提、DB 非接続、SQL 非実行、ファイルシステム非書き込み、未信頼 input 由来の signal は安全 contract として維持する。

## 0.10.5 の判断

5 tool すべてを 1.0.0 前の卒業対象とする。ただし、0.10.5 ではどの tool からも `experimental draft tool` marker を外さない。

0.10.5 は卒業実施ではなく、次の段階へ進む判断と残存 gap を固定する milestone とする。

- 0.11.0: tool ごとの卒業条件と全 response variant の要件を固定する。
- 0.12.0: stable output schema、draft safety contract、bounded output、MCP E2E、published-style validation を実装・強化する。
- 0.13.0: 条件を満たした tool から `experimental` marker を外し、1.0.0 final RC を行う。

## 判断したアプローチ

### 採用: 全5 tool を卒業予定とし、marker は維持する

5 tool はいずれも read-only draft helper として 1.0.0 の対象に含める価値がある。一方、stable output schema と全 tool の published validation が揃っておらず、Mermaid の成功 response には共通 draft safety contract の gap もある。このため、卒業方針だけを決定し、public contract の切り替えは0.12.0の hardening 後まで行わない。

### 不採用: 一部 tool を0.10.5で先行卒業させる

Mermaid 以外の tool にも stable output schema と全 variant の検証が不足している。先行卒業は tool 間で contract を分裂させ、0.10.5 の dry run と decision という責務を超えるため採用しない。

### 不採用: 全5 tool を1.0.0後まで experimental に残す

確認された gap は0.11.0から0.13.0で段階的に解消できる。卒業自体を1.0.0後へ送る必要はなく、既存 roadmap の段階移行を維持する。

| ----------------------------- | ---------------------- | ----------------------------------------------------------------------------------------------------- |

## 変更範囲

### Version lockstep

次の version surface を `0.10.5` に揃える。

- root、parser、core、CLI の package manifest
- MCP server metadata
- structured output の `schemaVersion`
- test expectation

version 以外の runtime behavior は変更しない。

### ROADMAP

- 0.10.5 を完了にする。
- 5 tool すべてを卒業予定とする判断を記録する。
- decision matrix と残存 gap を記録する。
- gap の修復先を0.11.0から0.13.0へ明示する。

### Public guidance

README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` に同じ意味の 0.10.5 decision を追加する。

- 5 tool は 1.0.0 前の卒業対象である。
- 0.10.5 では `experimental draft tool` marker を維持する。
- 卒業後も `draft` と安全 signal を維持する。
- contract hardening と marker removal は0.11.0から0.13.0で段階的に行う。

詳細な decision matrix は ROADMAP に集約し、公開ガイダンスへ重複させない。

### Regression guard

- docs regression test で3つの public guidance が 0.10.5 decision を同じ意味で説明することを確認する。
- smoke test で MCP server version と structured output の `schemaVersion` が `0.10.5` であることを確認する。
- 既存 test で5 tool の description に `experimental draft tool` が残り、output schema が未公開のままであることを確認する。

## Runtime contract

0.10.5 では次を変更しない。

- server の tool 登録内容
- tool profile の分類
- tool 名、tool 数、input schema
- tool description の `experimental draft tool` marker
- handler と generator の出力フィールド
- draft helper の MCP `outputSchema` 公開状態
- DB 非接続、SQL 非実行、ファイルシステム非書き込み、秘密情報マスク、untrusted content の境界

既存 client から見える runtime surface は、version metadata を除いて0.10.4と同じにする。

## TDD と検証

実装は `feat/0.10.5-draft-graduation-decision` worktree で行う。

1. `rtk pnpm agent:preflight` を実行する。
2. docs regression test に0.10.5 decision の期待値を追加し、既存 docs に対して意図した RED を確認する。
3. version expectation を `0.10.5` に更新し、既存 version surface に対して意図した RED を確認する。
4. version、ROADMAP、public guidance を最小限変更して GREEN にする。
5. focused docs / smoke tests を実行する。
6. CLI test suite を実行する。
7. `rtk pnpm release:check` を実行する。
8. `rtk pnpm published:check` を実行する。
9. git diff、status、秘密情報、ユーザー固有 path、無関係な変更の有無を確認する。

## 失敗時の扱い

- docs または version の drift は0.10.5で修正する。
- `release:check` または `published:check` の失敗は完了 blocker とし、原因を特定する。
- 既存 release gate を壊していない draft helper の runtime contract gap は0.10.5へ混ぜず、0.11.0または0.12.0の作業として残す。
- 0.10.4由来の release-blocking regression が見つかった場合は0.10.5完了を宣言せず、原因と修正範囲を再提示する。

## 完了条件

- 全 version surface が `0.10.5` で一致する。
- 5 tool すべての卒業判断と残存 gap が ROADMAP で説明できる。
- README、AGENTS.md、repo skill が同じ graduation 方針を示す。
- `experimental draft tool` marker と runtime contract が維持されている。
- focused tests、CLI test suite、`rtk pnpm release:check`、`rtk pnpm published:check` が通る。
- 秘密情報、ユーザー固有 path、無関係な変更が差分に含まれない。
- feature branch へコミットする。明示依頼がないため merge と push は行わない。

## Non-goals

- 0.10.5で `experimental` marker を外さない。
- draft helper の output schema や runtime output fields を追加・変更しない。
- Mermaid の draft safety contract gap を0.10.5で修正しない。
- 新しい tool、Resource、profile、ORM、framework 対応を追加しない。
- DB 接続、SQL 実行、ファイル書き込み、資格情報の復号・表示を追加しない。
