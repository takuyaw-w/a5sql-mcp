# a5sql-mcp 0.9.12 Published Package / MCP Client Matrix Design

## Goal

`0.9.12: Published Package / MCP Client Matrix` として、source tree ではなく install 後の package から起動した MCP server が、1.0.0 前の代表 contract を MCP client 経由で満たすことを検出できるようにする。

0.9.12 は新しい MCP tool や output schema を追加する milestone ではない。既存の `published:check` を、tarball install 後の CLI bin から起動する package 形態の validation matrix として強化する。

## Background

現在の実装 version は root workspace、`packages/parser`、`packages/core`、`packages/cli`、`A5SQL_MCP_SERVER_VERSION`、version assertion test がすべて `0.9.11` で揃っている。`ROADMAP.md` では 0.9.12 が `Published Package / MCP Client Matrix` として未完了になっている。

既存の `packages/cli/scripts/published-check.mjs` は、parser / core / cli の local tarball を一時 workspace に install し、install 後の `a5sql-mcp` bin から MCP server を起動して `tools/list` の exact set を確認している。さらに hostile SQL asset に対する `search_a5sql_assets` の最小 adversarial assertion も持っている。

0.9.12 では、この検証を source tree の smoke test と重複させすぎず、package 形態で壊れやすい entrypoint、packed files、workspace dependency resolution、MCP client transport、代表 tool call contract に焦点を当てる。

## Scope

### In Scope

- `packages/cli/scripts/published-check.mjs` を拡張し、install 後の CLI bin から起動した MCP server に対して MCP client 経由の代表 matrix を実行する。
- matrix で `tools/list`、代表 `callTool`、root required error、secret masking、untrusted content signal、draft disclosure を確認する。
- hostile fixture は一時 directory に生成し、実在する credential、ユーザー固有 path、外部 registry 依存を増やさない。
- root workspace、parser、core、cli、MCP server metadata、version assertion test を `0.9.12` に lockstep 更新する。
- `ROADMAP.md` の 0.9.12 を完了状態に更新する。
- 必要なら README / AGENTS / repo skill の release check 説明を最小限更新する。
- 0.9.12 の検証として `rtk pnpm release:check` と `rtk pnpm published:check` を実行する。

### Out of Scope

- 新しい MCP tool / resource の追加。
- tool 名、description、input schema、stable read-only / experimental draft 分類の変更。
- public JSON contract の再設計や全 tool の exhaustive validation。
- DB 接続、SQL 実行、A5:SQL 設定書き込み、資格情報の復号や表示。
- npm registry へ publish した package の live install 検証。
- 0.9.13 の範囲である README / AGENTS / skill onboarding freeze。
- 1.0.0 tag / publish 作業。

## Design

0.9.12 は feature branch または worktree で実施する。implementation plan の Task 0 で `rtk pnpm agent:preflight` を実行し、`main` 上で直接実装しない。

中心となる変更は `packages/cli/scripts/published-check.mjs` の matrix 化である。既存の tarball pack、temporary workspace、local tarball override、installed CLI bin 起動、`StdioClientTransport`、`Client` 接続の流れは維持する。その同じ installed-package MCP client session で、代表 call を順番に実行する。

matrix は次の contract を確認する。

- `tools/list`: 期待する 22 tool の exact set を確認し、`Tools: (none)` や期待 tool 不足を失敗にする。
- `search_a5sql_assets`: hostile SQL asset を検索し、`contentIsUntrusted: true`、asset payload、secret masking、固定 `nextAction` を確認する。
- `read_a5sql_asset`: hostile SQL 本文を読み、password、Bearer token、URL query credential、ODBC password が mask されることを確認する。
- `parse_a5sql_asset`: hostile SQL asset を解析し、`found: true`、`parser: "sql-heuristic"`、`contentIsUntrusted: true`、`untrustedPayloadFields` を確認する。
- `generate_sql_select`: sample `.a5er` から draft SQL を生成し、`outputKind: "draft"`、`readOnly: true`、`writesToFileSystem: false`、`connectsToDatabase: false`、`executesSql: false`、`draftIsDerivedFromUntrustedInput: true`、`draftOutputFields` を確認する。
- root required error: root 未指定の `parse_a5sql_asset` または `read_a5sql_asset` で `code: "roots_required"`、`warnings: ["roots_required"]`、safe `message` / `nextAction` を確認する。

`published-check.mjs` は test runner ではないため、重い assertion framework は導入しない。小さな helper 関数で structured content の object 判定、field assertion、secret absence assertion、array membership assertion を行い、失敗時はどの matrix step が壊れたか分かる固定 error message を投げる。

success message は、既存の意味を保ちつつ matrix 化が分かる文言に更新する。例: `published:check passed with 22 tools and installed-package MCP client matrix from installed package bin`。

## Contract Rules

0.9.12 の matrix は package 形態の regression guard であり、source tree の in-memory smoke test の全内容を複製しない。source tree で保証している詳細 contract は維持し、published check では install 後 bin で代表 path が実際に動くことを確認する。

A5:SQL 由来の hostile payload は untrusted content としてのみ扱う。payload は検索結果や本文に masked form で現れてよいが、raw secret や prompt injection 文が trusted guidance として扱われてはならない。

生成補助 tool は review 用 draft のまま扱う。`generate_sql_select` の output は SQL 実行結果ではなく、`draftIsDerivedFromUntrustedInput: true` と `draftOutputFields` を持つ installed-package contract として確認する。

root 未指定時は既定候補を探索しない。`roots_required` の `message` と `nextAction` は固定 guidance とし、hostile asset path や payload を含めない。

## Error Handling

`published-check.mjs` の failure は、package install / startup / tool surface / matrix call のどこで壊れたかが分かる message にする。raw MCP response を丸ごと出力せず、必要な fixed metadata と missing field 名を中心に示す。

秘密情報 fixture の raw value は failure message に直接含めない。secret absence の assertion は、固定 secret labels の raw value が serialized response に含まれないことを確認し、失敗時は secret category だけを示す。

temporary workspace は現行通り `finally` で削除する。client close と temp directory cleanup は失敗時にも試みる。

## Testing

TDD で進める。最初に `0.9.12` version assertion と、`published-check.mjs` の matrix expectation を追加して失敗を確認する。次に version lockstep と matrix implementation を入れて focused check を green にする。

focused checks は次を想定する。

- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-server-smoke.test.ts`
- `rtk pnpm published:check`

必要に応じて package entrypoint 周辺の変更後に次も実行する。

- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-tools.test.ts`
- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-asset-tools.test.ts`

最終検証は次を実行する。

- `rtk pnpm release:check`
- `rtk pnpm published:check`

## Acceptance Criteria

- `A5SQL_MCP_SERVER_VERSION` と全 package version が `0.9.12` に揃っている。
- `published:check` が install 後の CLI bin から MCP server を起動し、期待 22 tool の exact set を確認している。
- `published:check` が MCP client 経由で代表 `callTool`、root required error、secret masking、untrusted content signal、draft disclosure を確認している。
- package 形態の hostile fixture assertion が、raw secret leak、`Tools: (none)`、期待 tool 不足、draft disclosure 退行を release 前に検出できる。
- registry や network への依存を増やさず、local tarball install の再現性を維持している。
- `ROADMAP.md` で 0.9.12 が完了状態になっている。
- `rtk pnpm release:check` と `rtk pnpm published:check` が成功する。
