# a5sql-mcp 0.8.0 Generation Tools Design

## Goal

0.8.0 では、生成系 tool を削らずに 1.0.0 の中核である「ローカル A5:SQL 資産の読み取り」と境界づける。実装上は `tool-outputs.ts` の肥大化を抑え、生成系 tool が返す内容を「レビュー用の案」として明示する。

## Scope

対象にする tool:

- `generate_sql_select`
- `generate_model_files`
- `generate_schema_markdown`
- `generate_migration_plan`

対象外:

- 新しい ORM / framework 対応の追加。
- DB への接続、SQL 実行、migration 適用。
- A5:SQL 設定ファイルや `.a5er` ファイルへの書き込み。
- 生成結果をファイルシステムへ保存する機能。

## Design

`packages/cli/src/mcp/tool-outputs.ts` は、既存の公開 export を保つ薄い集約ファイルに寄せる。生成系の出力整形は新しい `packages/cli/src/mcp/generation-tools.ts` に移し、既存 handler や test import は壊さない。

`generation-tools.ts` は次の責務を持つ。

- SQL SELECT 案の生成。
- model file 案の生成。
- schema Markdown 案の生成。
- migration plan 案の生成。
- 生成系 tool 共通の disclosure を構造化フィールドとして付与する。

生成系 tool のレスポンスには、既存の `nextAction` や `warnings` を残したまま、少なくとも次の情報を追加する。

- `outputKind`: `"draft"`。
- `readOnly`: `true`。
- `writesToFileSystem`: `false`。
- `connectsToDatabase`: `false`。
- `executesSql`: `false`。

`generate_migration_plan` では、`includeDestructive` のデフォルトを `false` のまま固定する。`includeDestructive: false` のとき、DROP 系 operation は返さず warning に留める。`includeDestructive: true` のときだけ `destructive: true` operation を返す。

## Documentation

README には、生成系 tool は read-only の生成補助であり、1.0.0 の中核読み取り API とは別のレビュー支援であることを書く。ORM / migration framework 対応はむやみに増やさず、Laravel Eloquent、SQLAlchemy、plain SQL / Laravel / Alembic の既存範囲を維持する。

AGENTS.md と `.agents/skills/a5sql-mcp/SKILL.md` は、既に生成系 tool を「案」として説明しているため、必要な場合だけ wording を補強する。

## Testing

TDD で次の behavior を固定する。

- `generate_model_files` は draft disclosure を返し、ファイルシステムには書き込まない契約を構造化フィールドで示す。
- `generate_schema_markdown` は draft disclosure を返す。
- `generate_sql_select` は draft disclosure を返し、SQL を実行しない契約を示す。
- `generate_migration_plan` は draft disclosure を返し、`includeDestructive` 省略時に DROP operation を返さない。
- `includeDestructive: true` の場合だけ destructive operation を返す既存挙動を維持する。
- `tool-outputs.ts` 分割後も既存の public import が壊れない。

最終検証は `pnpm release:check` とする。
