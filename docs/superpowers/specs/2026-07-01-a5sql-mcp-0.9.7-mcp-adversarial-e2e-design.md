# a5sql-mcp 0.9.7 MCP Adversarial E2E Design

## Goal

0.9.7 では、handler 直接テストだけでなく、MCP server 起動後の `tools/list` / `callTool` 経由で安全 contract を確認する。目的は新しい tool を増やすことではなく、MCP クライアントから見える public surface で secret masking、untrusted content signal、draft disclosure、root boundary が崩れていないことを release 前に検出できるようにすることである。

## Scope

対象にするもの:

- InMemory MCP client を使った `tools/list` と代表 `callTool` の adversarial E2E。
- hostile SQL / secret を含む synthetic fixture root を使った `search_a5sql_assets`、`read_a5sql_asset`、`parse_a5sql_asset` の確認。
- 代表 draft tool の disclosure field 確認。
- root 未指定または root 外 path の error contract が MCP client 経由でも固定 guidance だけを返すことの確認。
- published-style startup check に、最低限の adversarial assertion を追加できる範囲で追加する。
- root workspace、parser、core、cli、MCP server metadata の version lockstep。
- README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` の release candidate / 0.9.7 検証説明。

対象外:

- 新しい MCP tool の追加。
- A5:ER parser の再設計、SQL parser の機能追加、root 解決ロジックの仕様変更。
- 接続先 DB への SQL 実行、A5:SQL 設定ファイルへの書き込み、資格情報の復号や表示。
- 0.9.8 で扱う client / agent safety docs の大幅な書き換え。
- 0.9.9 で扱う API freeze の全面棚卸し。

## Design

既存の `packages/cli/test/mcp-server-smoke.test.ts` を、単なる smoke test から MCP adversarial E2E に拡張する。テストは `createA5sqlMcpServer` と MCP SDK の `InMemoryTransport` を使い、実際の MCP client API で `listTools` と `callTool` を呼ぶ。handler 直接呼び出しではなく、client から見える `structuredContent` を検証対象にする。

fixture はテストごとの一時ディレクトリに作る。起動時ファイルには小さな `.a5er` を使い、asset root には hostile SQL を置く。hostile SQL には prompt injection 風コメント、JSON credential、Bearer token、URL query credential、ODBC 形式 credential を含める。これにより、`search_a5sql_assets` と `read_a5sql_asset` の snippet / content masking、`parse_a5sql_asset` の `contentIsUntrusted` と output boundary field を MCP client 経由で確認できる。

root boundary は、MCP client から `read_a5sql_asset` または `parse_a5sql_asset` を root 未指定で呼び、`roots_required` または `asset_path_requires_roots` の固定 error contract を確認する。error response の `message`、`warnings`、`nextAction` は A5:SQL 由来 payload を含まない固定 guidance として扱う。

draft disclosure は、代表として `generate_sql_select` を MCP client から呼び、`outputKind: "draft"`、`readOnly: true`、`writesToFileSystem: false`、`connectsToDatabase: false`、`executesSql: false`、`draftIsDerivedFromUntrustedInput: true` を確認する。生成結果そのものの SQL 文面は review 用 draft payload として扱い、安全性の検証は disclosure field に寄せる。

`packages/cli/scripts/published-check.mjs` は、install 後の bin から `tools/list` を確認している既存検証を維持する。そのうえで temp workspace 内に hostile SQL asset root を作り、installed package の MCP server に対して最低限 `search_a5sql_assets` または `read_a5sql_asset` を呼ぶ。ここでは detailed contract を重複させすぎず、raw secret が返らないこと、`contentIsUntrusted: true` が返ること、期待 tool 不足が失敗になることを確認する。

version は root workspace、parser、core、cli、MCP server metadata で `0.9.7` に揃える。README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` は、0.9.7 の release candidate 検証として MCP client 経由の adversarial E2E と `published:check` の位置づけを短く更新する。

## Testing

TDD で次を固定する。

- `tools/list` に期待 tool が含まれ、代表生成 tool の description に `experimental draft` が含まれる。
- MCP client 経由の `search_a5sql_assets` は hostile SQL asset の snippet を返しても raw secret を返さず、`contentIsUntrusted: true`、`trustedMetadataFields`、`untrustedPayloadFields` を返す。
- MCP client 経由の `read_a5sql_asset` は本文を返しても password、token、Bearer token、ODBC password を mask する。
- MCP client 経由の `parse_a5sql_asset` は SQL comment / statement を untrusted payload として返し、trusted guidance に prompt injection 文を混ぜない。
- MCP client 経由の root 未指定 error は、固定 `code` / `warnings` / `message` / `nextAction` を返し、ローカル path や hostile payload を含まない。
- MCP client 経由の `generate_sql_select` は draft disclosure field を返し、DB 接続、SQL 実行、ファイル書き込みをしない contract を示す。
- published-style startup check は installed package bin で `tools/list` と最小 adversarial assertion を確認する。
- version は root workspace、parser、core、cli、MCP server metadata で `0.9.7` に揃っている。

最終検証は `rtk pnpm release:check` と `rtk pnpm published:check` とする。

## Success Criteria

- MCP クライアント経由でも、mask、untrusted content signal、draft disclosure、root boundary が崩れていない。
- `Tools: (none)`、期待 tool 不足、代表 tool の contract regression を release 前に検出できる。
- published-style startup check が、単なる `tools/list` だけでなく、最低限の adversarial contract regression も検出する。
- 0.9.7 の docs と version metadata が実装済み検証内容と一致している。
- DB 接続、SQL 実行、ファイル書き込み、資格情報復号は引き続き non-goal として維持されている。
