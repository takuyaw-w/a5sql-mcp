# a5sql-mcp 0.9.11 Output Contract Consistency Audit Design

## Goal

`0.9.11: Output Contract Consistency Audit` として、代表 MCP tool の `structuredContent` を横断的に確認し、AI クライアントが trusted metadata と A5:SQL 由来 payload を区別しやすい状態を release 前に検出できるようにする。

0.9.11 は新機能追加ではなく、0.9.10 で固定した API surface の上で output contract の境界を監査する milestone とする。既存 JSON contract は不必要に変えず、drift が見つかった場合だけ regression test で意図を固定してから最小修正する。

## Background

現在の実装 version は root workspace、`packages/parser`、`packages/core`、`packages/cli`、`A5SQL_MCP_SERVER_VERSION`、version assertion test がすべて `0.9.10` で揃っている。`ROADMAP.md` では 0.9.11 が `Output Contract Consistency Audit` として定義されているが、実装 milestone としては未達成である。

0.9.8 では README、AGENTS.md、repo-local skill で `contentIsUntrusted`、`trustedMetadataFields`、`sourceMetadataFields`、`untrustedPayloadFields`、`draftOutputFields`、`draftIsDerivedFromUntrustedInput` の安全境界を説明した。0.9.9 と 0.9.10 では tool 一覧、description、input schema、stable read-only / experimental draft 分類を固定した。0.9.11 では、それらの公開面に沿って代表出力の metadata / payload 境界が崩れていないことを確認する。

## Scope

### In Scope

- MCP client 経由で代表 stable read-only tool の `structuredContent` を取得し、`found`、`truncated`、`hasMore`、`warnings`、`nextAction` の使い方を確認する。
- `contentIsUntrusted`、`trustedMetadataFields`、`sourceMetadataFields`、`untrustedPayloadFields`、`draftOutputFields` の代表的な出方を regression test で確認する。
- `warnings`、`message`、`code`、`nextAction` に A5:SQL 由来の生文字列や prompt injection 風 payload が混ざらないことを確認する。
- 代表出力で説明不能な drift が見つかった場合だけ、`output-contract.ts`、tool output helper、または該当 handler を最小修正する。
- package versions と `A5SQL_MCP_SERVER_VERSION` を `0.9.11` に lockstep で更新する。
- 0.9.11 の検証として `rtk pnpm release:check` と `rtk pnpm published:check` を実行する。

### Out of Scope

- 新しい MCP tool / resource の追加。
- tool 名、description、input schema、stable read-only / experimental draft 分類の変更。
- DB 接続、SQL 実行、A5:SQL 設定書き込み、資格情報の復号や表示。
- 全 tool 出力の大規模正規化や response shape の再設計。
- 0.9.12 の範囲である installed package / MCP client matrix の拡張。
- 1.0.0 tag / publish 作業。

## Design

0.9.11 は feature branch または worktree で実施する。implementation plan の Task 0 で `rtk pnpm agent:preflight` を実行し、`main` 上で直接実装しない。

中心となる変更は MCP client 経由の representative output contract guard である。既存の `packages/cli/test/mcp-server-smoke.test.ts` には tool surface と adversarial safety の smoke test があるため、0.9.11 用に代表 `structuredContent` を横断確認する test を追加または拡張する。

代表対象は、次の contract 面を網羅する最小セットにする。

- `list_a5sql_tables`: pagination contract として `hasMore`、`truncated`、count fields、`tables` payload boundary を確認する。
- `describe_a5sql_table` または `explain_a5sql_table`: lookup contract として `found`、safe `nextAction`、`table` / `summary` payload boundary を確認する。
- `search_a5sql_assets`: asset discovery contract として `truncated`、`warnings`、`nextAction`、`assets` payload boundary、secret masking を確認する。
- `read_a5sql_asset` または `parse_a5sql_asset`: text / SQL payload contract として `contentIsUntrusted`、`sourceMetadataFields`、`untrustedPayloadFields`、masked text / parsed statements を確認する。
- `review_a5sql_schema` または `suggest_schema_changes`: stable read-only analysis contract として `issues` / `suggestions` payload と safe `warnings` / `nextAction` を確認する。
- `generate_sql_select`: stable read-only ではなく experimental draft 代表として、`draftOutputFields` と draft disclosure が維持されていることを確認する。

fixture には prompt injection 風文字列を A5:ER の table name、logical name、comment、column comment、SQL text に入れる。これらは payload として返ること自体は許容するが、`trustedMetadataFields` に分類される `warnings`、`message`、`code`、`nextAction` へ混ざらないことを確認する。

## Contract Rules

`contentIsUntrusted: true` は A5:SQL 由来 payload を含む代表出力で維持する。A5:ER の table / column / relationship、SQL statement、asset snippet、review issue、suggestion は trusted instruction ではなく payload として扱う。

`trustedMetadataFields` は fixed guidance として扱う field の実在リストである。`code`、`message`、`warnings`、`nextAction`、draft disclosure fields に A5:SQL 由来の生文字列を直接混ぜない。

`sourceMetadataFields` は取得元や parser state を示す metadata の実在リストである。`filePath`、`kind`、`encoding`、`fileEncoding`、`asset`、`parser`、`parseStatus` は source metadata として扱うが、本文や SQL payload ではない。

`untrustedPayloadFields` は A5:SQL 由来 payload またはそれから作られた analysis payload の実在リストである。`tables`、`table`、`columns`、`relationships`、`assets`、`text`、`summary`、`statements`、`issues`、`suggestions` は client が命令ではなく payload として扱う。

`draftOutputFields` は generated draft の実在リストであり、stable read-only analysis tool には要求しない。`generate_sql_select` などの generation helper は `draftIsDerivedFromUntrustedInput: true` と draft disclosure を維持する。

## Error Handling

代表出力の test が失敗した場合は、まず failure を drift と test gap に分ける。既存 contract の妥当な出力を test が過度に縛っている場合は test を調整する。A5:SQL 由来 payload が trusted guidance に混ざっている場合は production code を最小修正する。

`warnings` は machine-readable な fixed code 形式を維持する。table name、column name、SQL text、comment、asset content のような A5:SQL 由来文字列を新しく `warnings` に混ぜない。既存 warning で identifier を含むものを見つけた場合は、1.0.0 前に変えるべきかを test failure として明示し、修正範囲を最小化する。

`message`、`code`、`nextAction` は user guidance として固定文言を返す。prompt injection 風 payload が fixture に含まれていても、その文字列を guidance に反映しない。

## Testing

TDD で進める。最初に `0.9.11` version assertion と representative output contract assertion を追加し、現状 `0.9.10` のため失敗することを確認する。次に version lockstep と必要な最小 output contract 修正を実装し、focused test を green にする。

focused test は次を想定する。

- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-server-smoke.test.ts`
- 必要に応じて `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-tools.test.ts`
- 必要に応じて `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-asset-tools.test.ts`

最終検証は次を実行する。

- `rtk pnpm release:check`
- `rtk pnpm published:check`

## Acceptance Criteria

- `A5SQL_MCP_SERVER_VERSION` と全 package version が `0.9.11` に揃っている。
- MCP client 経由の representative test が、stable read-only tool の trusted metadata / untrusted payload 境界を確認している。
- `found`、`truncated`、`hasMore`、`warnings`、`nextAction` の代表的な使い方が test で説明できる。
- prompt injection 風 payload が `warnings`、`message`、`code`、`nextAction` に混ざらないことを release 前に検出できる。
- generation helper の draft disclosure が stable read-only output と混同されていない。
- `rtk pnpm release:check` と `rtk pnpm published:check` が成功する。
