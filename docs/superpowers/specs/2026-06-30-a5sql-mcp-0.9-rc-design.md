# a5sql-mcp 0.9.0 Release Candidate Design

## Goal

0.9.0 では、1.0.0 に向けた release candidate として公開 API、ドキュメント、release workflow の再現性を固定する。新しい MCP tool は追加せず、0.8.0 時点の読み取り専用 API と生成補助 API が npm package 形態でも起動できることを確認できるようにする。

## Scope

対象にするもの:

- package tarball からの published-style MCP startup 検証。
- `tools/list` に期待 tool が出ることの検証。
- README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` の tool 一覧と 1.0.0 non-goal の整合。
- root workspace、parser、core、cli、MCP server metadata の version lockstep 更新。
- `release:check`、`pack:check`、tag/version preflight の利用手順の明確化。

対象外:

- 新しい MCP tool の追加。
- 既存 tool 名や入力/出力 schema の破壊的変更。
- DB への接続、SQL 実行、A5:SQL 設定ファイルや `.a5er` ファイルへの書き込み。
- 資格情報の復号、表示、保存。
- Web UI、常駐 daemon、追加 ORM / migration framework 対応。

## Design

0.9.0 の中心は、source tree 上の unit / in-memory test ではなく、package としての起動経路を検証することである。CLI package を build / pack し、一時ディレクトリに install して、install 後の bin から `--mcp <file>` で stdio MCP server を起動する検証スクリプトを追加する。

検証スクリプトは repo 内の sample `.a5er` を使い、MCP client から `tools/list` を呼び出す。`.a5er` 起動時に登録される tool を含め、現在公開している tool 名が揃っていることを確認する。`Tools: (none)` 相当の状態、期待 tool の不足、server 起動失敗、client 接続失敗は失敗として扱う。

既存の `packages/cli/test/mcp-server-smoke.test.ts` は、in-memory transport で version metadata、代表 tool、代表 `structuredContent` の契約を引き続き守る。published-style 検証はそれとは別に、npm package と bin entrypoint の壊れ方を検出する役割を持つ。

`pnpm release:check` は既存の format、lint、build、test、typecheck、pack dry-run を保つ。published-style 検証は、一時 install と MCP process 起動を含むため `release:check` には組み込まず、`pnpm published:check` のような明示コマンドとして追加する。0.9.0 RC の検証手順では、`pnpm release:check` と published-style startup 検証コマンドの両方を実行する。

Version は root workspace、`packages/parser`、`packages/core`、`packages/cli`、`A5SQL_MCP_SERVER_VERSION` を `0.9.0` に揃える。publish workflow の tag preflight は `v0.9.0` tag と package versions の一致を確認する既存方針を維持する。

## Documentation

README には 0.9.0 RC の検証手順を書く。利用者向け install / startup 例は、`npx -y @takuyaw-w/a5sql-mcp --mcp /absolute/path/to/model.a5er` を中心に維持する。

README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` では、次を一貫して明記する。

- ローカル A5:SQL 資産を読み取り専用で扱う。
- 接続先 DB への SQL 実行は 1.0.0 まで non-goal。
- A5:SQL 設定ファイルや ER 図ファイルへの書き込みは non-goal。
- 資格情報の復号、表示、保存は non-goal。
- 生成系 tool は review 用の案生成であり、DB やファイルシステムに副作用を持たない。

tool 一覧は、README / AGENTS.md / skill / server registration のずれを最小にする。0.9.0 では一覧の生成までは行わず、公開 tool 名の期待値を検証側に固定する。

## Testing

TDD で次の behavior を固定する。

- `mcp-server-smoke.test.ts` は `0.9.0` の server metadata と package versions を確認する。
- smoke test は代表 tool の `structuredContent` が契約フィールドと secret masking を守ることを確認する。
- published-style 検証は tarball install 後の bin から MCP server を起動し、`tools/list` が期待 tool を返すことを確認する。
- published-style 検証は `example/schema.a5er` などの repo-local fixture を使い、ユーザー固有パスや秘密情報を fixture に含めない。
- docs に書いた RC 検証コマンドが実際に実行できることを確認する。

最終検証は `pnpm release:check` と published-style startup 検証コマンドとする。

## Success Criteria

- npm package tarball から MCP server を起動し、期待 tool 一覧を取得できる。
- README / AGENTS.md / skill の tool 一覧と non-goal 表現が 0.9.0 の公開面と一致している。
- package versions と MCP server version が `0.9.0` で揃っている。
- `pnpm release:check` と published-style startup 検証が通る。
- 1.0.0 に入れない項目が README から明確に分かる。
