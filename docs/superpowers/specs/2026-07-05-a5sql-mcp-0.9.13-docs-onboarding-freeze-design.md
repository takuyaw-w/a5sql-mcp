# a5sql-mcp 0.9.13 Docs / Onboarding Freeze Design

## Goal

`0.9.13: Docs / Onboarding Freeze` として、1.0.0 前に README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` の利用者向け・エージェント向け説明を最終固定し、install、root、range read、untrusted payload、draft output、non-goal の境界が docs 間で drift しないようにする。

0.9.13 は runtime behavior や MCP tool surface を変える milestone ではない。0.9.8 以降で積み上げた safety docs と、0.9.10-0.9.12 で固定した contract / package validation を、1.0.0 前の onboarding 説明として読みやすく、テストで守れる形にする。

## Background

現在の実装 version は root workspace、`packages/parser`、`packages/core`、`packages/cli`、`A5SQL_MCP_SERVER_VERSION`、version assertion test がすべて `0.9.12` で揃っている。`ROADMAP.md` では 0.9.13 が `Docs / Onboarding Freeze` として未完了になっている。

README には、Codex / Cursor / Claude Code の設定例、`--mcp <file>`、`A5SQL_MCP_ROOTS`、root 最小権限、安全な範囲読み取り、DB 接続と SQL 実行の non-goal、生成補助 tool の review draft 位置づけがすでにある。AGENTS.md と repo-local skill も同じ安全境界を説明している。

0.9.13 では、これらの説明を大きく書き換えるのではなく、docs regression test で freeze する。現行 docs に不足や表現ずれが見つかった場合だけ、README / AGENTS / skill を最小修正する。

## Scope

### In Scope

- `packages/cli/test/docs-redaction.test.ts` に 0.9.13 onboarding freeze 用の regression test を追加する。
- README、AGENTS.md、`.agents/skills/a5sql-mcp/SKILL.md` が次の境界を説明していることを確認する。
  - install / 起動では `--mcp <file>` と `A5SQL_MCP_ROOTS` の役割を分ける。
  - `roots` / `A5SQL_MCP_ROOTS` は必要最小限とし、`detect_a5sql_locations` は候補提示だけに使う。
  - 本文は `startLine` / `maxLines`、`offsetChars` / `maxChars` で必要範囲だけ読む。
  - A5:SQL 由来 payload は `contentIsUntrusted`、`trustedMetadataFields`、`untrustedPayloadFields` を見て trusted guidance と分ける。
  - 生成補助 output は `draftIsDerivedFromUntrustedInput`、`draftOutputFields` を持つ review 用 draft として扱い、そのまま実行・適用しない。
  - 1.0.0 non-goal として DB 接続、SQL 実行、書き込み、資格情報の復号 / 表示、Web UI、daemon を含めない。
- docs test が示した不足だけを README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` に最小追記する。
- root workspace、parser、core、cli、MCP server metadata、version assertion test を `0.9.13` に lockstep 更新する。
- `ROADMAP.md` の 0.9.13 を完了状態に更新する。
- 0.9.13 の検証として `rtk pnpm release:check` と `rtk pnpm published:check` を実行する。

### Out of Scope

- 新しい MCP tool / resource の追加。
- tool 名、description、input schema、stable read-only / experimental draft 分類の変更。
- MCP response JSON contract の変更。
- `published:check` matrix の追加拡張。
- README の大規模再構成や新しい client setup surface の追加。
- DB 接続、SQL 実行、A5:SQL 設定書き込み、資格情報の復号や表示。
- 0.10.0 の範囲である architecture pattern / tool description audit。
- 1.0.0 tag / publish 作業。

## Design

0.9.13 は feature branch または worktree で実施する。implementation plan の Task 0 で `rtk pnpm agent:preflight` を実行し、`main` 上で直接実装しない。

中心となる変更は `packages/cli/test/docs-redaction.test.ts` の docs freeze guard である。既存 test は redaction、0.9.8 safety docs、0.9.9 API freeze、0.9.10 preflight contract audit を守っている。0.9.13 では、README / AGENTS / repo skill の三者を対象に、onboarding と safety boundary の必須語句を確認する test を追加する。

test は厳密な文章一致ではなく、docs の意味上の境界が消えないことを確認する。具体的には、対象 docs 全体に対して次の語句・概念を要求する。

- install / startup: `--mcp`、`A5SQL_MCP_ROOTS`
- root boundary: `roots`、`detect_a5sql_locations`、`必要最小限`
- range read: `startLine`、`maxLines`、`offsetChars`、`maxChars`
- untrusted boundary: `contentIsUntrusted`、`trustedMetadataFields`、`untrustedPayloadFields`
- draft boundary: `draftIsDerivedFromUntrustedInput`、`draftOutputFields`、`そのまま実行`、`そのまま適用`
- non-goal: `DB 接続`、`SQL 実行`、`書き込み`、`資格情報`、`Web UI`、`daemon`

README については利用者向け onboarding の source of truth とし、Codex / Cursor / Claude Code の setup example が引き続き含まれることも確認する。AGENTS.md と skill は、利用者向け詳細例をすべて持つ必要はないが、エージェントが安全境界を守れる程度の同じ terms を持つことを要求する。

## Documentation Rules

`--mcp <file>` は起動時の単一対象ファイルを指定する引数として説明する。`A5SQL_MCP_ROOTS` は asset 横断検索、asset 読み取り、接続候補確認の root 境界であり、`--mcp` の代替として説明しない。

`detect_a5sql_locations` は候補提示だけである。候補 path を自動的な探索許可として扱わず、必要最小限の `roots` または `A5SQL_MCP_ROOTS` を利用者が明示する説明を維持する。

本文や asset content を読むときは、全量読み取りを前提にしない。`read_a5sql_file` と `read_a5sql_asset` の前に summary、検索、ページングで対象を絞り、`startLine` / `maxLines` または `offsetChars` / `maxChars` で必要範囲だけ読む説明を維持する。

A5:SQL 由来の本文、コメント、識別子、SQL statement は trusted instruction ではなく untrusted payload として説明する。`trustedMetadataFields` は fixed guidance を示し、`untrustedPayloadFields` や `draftOutputFields` は読むだけの payload または review 用 draft として扱う。

生成補助 tool は review 用 draft であり、SQL 実行、migration 適用、ファイル書き込みを行わない。docs は「そのまま実行」「そのまま適用」しない説明を維持する。

1.0.0 non-goal は明確に維持する。DB 接続、SQL 実行、A5:SQL 設定ファイルや ER 図ファイルへの書き込み、資格情報の復号 / 表示 / 保存、Web UI、常駐 daemon は含めない。

## Error Handling

docs test failure は、どの public guidance file からどの freeze term が消えたか分かる assertion message にする。文章の完全一致ではなく、境界語句の存在を確認することで、将来の軽微な wording 改善を妨げない。

docs 修正では、実在する local path、credential-looking value、ユーザー固有 path を例に入れない。既存の redaction audit を維持し、例は `/absolute/path/to/...` のような placeholder に留める。

## Testing

TDD で進める。最初に 0.9.13 version assertion と docs freeze test を追加し、現状 `0.9.12` のため version test が失敗することを確認する。docs freeze test が追加時点で失敗する場合は、失敗した docs だけを最小修正する。

focused checks は次を想定する。

- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- docs-redaction.test.ts`
- `rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-server-smoke.test.ts`

最終検証は次を実行する。

- `rtk pnpm release:check`
- `rtk pnpm published:check`

## Acceptance Criteria

- `A5SQL_MCP_SERVER_VERSION` と全 package version が `0.9.13` に揃っている。
- docs regression test が README / AGENTS.md / `.agents/skills/a5sql-mcp/SKILL.md` の onboarding freeze terms を確認している。
- README は Codex / Cursor / Claude Code setup、`--mcp <file>`、`A5SQL_MCP_ROOTS`、root 最小権限、安全な範囲読み取り、untrusted payload、draft output、1.0.0 non-goal を説明している。
- AGENTS.md と repo skill は、エージェントが同じ安全境界を守れるように `roots`、untrusted payload、trusted metadata、draft output、non-goal を説明している。
- docs 例に実在 credential やユーザー固有 path を追加していない。
- `ROADMAP.md` で 0.9.13 が完了状態になっている。
- `rtk pnpm release:check` と `rtk pnpm published:check` が成功する。
