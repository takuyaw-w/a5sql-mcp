# a5sql-mcp 0.9.3 Trusted Metadata / Untrusted Payload Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add machine-readable output boundary classification for trusted metadata, source metadata, untrusted payload, and draft output.

**Architecture:** Preserve the existing MCP JSON payload shape and add top-level classification fields. Centralize the classification logic in `packages/cli/src/mcp/output-contract.ts`, then wrap read-only output and generation draft output through that helper.

**Tech Stack:** TypeScript, Vitest, pnpm workspace, MCP TypeScript SDK.
