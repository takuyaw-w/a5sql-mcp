# a5sql-mcp 0.10.4 MCP Resource Gateway Pilot Implementation Plan

Status: approved

## Goal

承認済み design に従い、起動時ファイル metadata と bounded A5:ER schema summary を固定 URI の read-only MCP Resource として追加する。path privacy、secret masking、untrusted content、既存 tool/profile contract を維持し、version 0.10.4 として release gate まで検証する。

実装は `feat/0.10.4-resource-gateway` branch で行い、`main` / `master` 上では実装しない。

## Task 2: Resource payload builder と registration

対象:

- 新規 `packages/cli/src/mcp/resources.ts`
- `packages/cli/src/mcp/server.ts`
- Resource unit tests

Implementation:

- Resource name/URI/MIME constants を定義する。
- `createConfiguredFileSummaryPayload()` を pure builder として追加する。
- `createConfiguredSchemaSummaryPayload()` を pure builder として追加する。
- A5:ER table/relationship summary を各 20 件に制限する。
- payload から `filePath`、basename、root を除外する。
- `.a5er` の場合だけ schema summary を登録する。
- `registerA5sqlResources()` を `createA5sqlMcpServer()` から呼ぶ。
- Resource 登録を `toolProfile` から独立させる。
- `getParsedFile` cache と read-time `stat` で freshness を維持する。

Verification:

```bash

```

## Task 3: 共通 public masking boundary

対象:

- 新規 `packages/cli/src/mcp/public-output.ts`
- `packages/cli/src/mcp/tool-handlers.ts`
- `packages/cli/src/mcp/resources.ts`
- masking regression tests

Red tests:

- schema comment 内の password/token が Resource text に残らない。
- URL userinfo、query token、ODBC password、Authorization header が mask される。
- secret-like path segment と起動時 absolute path が Resource に残らない。
- existing `read_a5sql_file` / `read_a5sql_asset` masking が変わらない。

Implementation:

- `maskForPublicConsumption()` と query-secret recovery helper を tool handler から共通 module に抽出する。
- Resource payload は `JSON.stringify` 後に同じ masking helper を通す。
- path は masking に依存せず payload construction 時点で除外する。
- raw exception message を Resource content に含めない。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-resources.test.ts mcp-asset-tools.test.ts mcp-server-smoke.test.ts
```

## Task 4: Untrusted content と bounded summary contract

対象:

- `packages/cli/src/mcp/resources.ts`
- Resource contract tests

Red tests:

- prompt injection 風 table/comment payload が `contentIsUntrusted: true` で返る。
- `untrustedPayloadFields` に `tables`、`relationships`、`warningDetails` が含まれる。
- fixed `warnings` / `nextAction` に A5:SQL 由来 payload が混ざらない。
- 21 件以上の table/relationship は 20 件で切られ、returned count と `truncated` が一致する。
- configured file summary は本文を含まず `contentIsUntrusted: false` である。

Implementation:

- trusted/source/untrusted field classification を Resource payload に含める。
- existing summary helper の安全な変換を再利用しつつ、path を明示的に omit する。
- arbitrary client input で上限を変更する Resource template は追加しない。
- deeper read の `nextAction` は既存 tool 名を固定 guidance として返す。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-resources.test.ts
```

## Task 5: Fixed Resource error contract と freshness

対象:

- `packages/cli/src/mcp/resources.ts`
- Resource tests

Red tests:

- server 起動後の file size/mtime 変更が次回 Resource read に反映される。
- A5:ER 内容変更後に schema summary が再 parse される。
- file 削除後に固定 `configured_file_unavailable` payload を返す。
- error payload に absolute path、OS error message、secret がない。
- unknown Resource URI は SDK の protocol error になる。

Implementation:

- callback 内の stat/parse error を固定 Resource error payload に変換する。
- `retryable: true` と固定 `nextAction` を返す。
- stack、errno detail、exception message を serialise しない。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-resources.test.ts
```
