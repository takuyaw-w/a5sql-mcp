# a5sql-mcp 0.10.2 Contract Integrity Implementation Plan

Status: approved

## Goal

承認済み design に従い、既存 22 tool の trusted / untrusted 境界、bounded asset lookup、partial parse、SQL statement count、A5:ER scalar fidelity、structured error、safe observability を TDD で修正する。

実装は `feat/0.10.2-contract-integrity` の隔離 worktree で行う。`main` / `master` 上では実装しない。

## Task 0: Agent preflight

実装開始前に次を実行する。

```bash
rtk pnpm agent:preflight
rtk git status --short --branch
```

確認事項:

- `main` / `master` ではない。
- detached HEAD ではない。
- worktree が clean である。
- branch が `feat/0.10.2-contract-integrity` である。
- preflight failure を `--allow-main` で回避しない。

現在の design / plan 文書は plan 承認前の review artifact である。plan 承認後、文書を planning checkpoint として扱ってから、実装開始時点で clean な worktree にして preflight を実行する。

## Task 1: Parser warning と scalar fidelity

対象:

- `packages/parser/src/types.ts`
- `packages/parser/src/index.ts`
- `packages/parser/test/a5er.test.ts`

Red tests:

- encoding mismatch の実値が `warnings` に入らない。
- table section 名、relationship 名が `warnings` に入らない。
- 可変値が `warningDetails` に入る。
- `9007199254740993` と負の BIGINT が文字列で保持される。
- 高精度 decimal が文字列で保持される。
- safe integer は number のままになる。
- position、relation type、key order は既存どおり number になる。

Implementation:

- `ParsedA5erWarningDetail` を追加する。
- `ParsedA5erDocument` に `warningDetails` を追加する。
- `warnings` を固定 code にする。
- `parseScalar()` は safe integer だけを number 化する。
- unsafe integer と decimal は文字列で保持する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp-parser test
```

## Task 2: SQL statement metadata

対象:

- `packages/parser/src/sql.ts`
- `packages/parser/src/types.ts`
- `packages/parser/src/index.ts`
- `packages/parser/test/sql.test.ts`
- `packages/cli/src/index.ts`
- CLI parser tests

Red tests:

- 101 statements が 100 件に丸められない。
- `maxStatements: 1000` で 1,000 件返る。
- 1,001 件で total 1,001、returned 1,000、truncated true になる。
- quote、line comment、block comment、dollar quote 内の semicolon を誤分割しない。
- source truncation 時に末尾 statement の不完全可能性を示す。
- configured file と asset parse で count semantics が一致する。

Implementation:

- 隠れた `.slice(0, 100)` を削除する。
- metadata 付き SQL parse API を追加する。
- `parseSqlStatements()` は compatibility API として維持する。
- `totalStatementCount`、`returnedStatementCount`、`statementsTruncated`、`trailingStatementMayBeIncomplete` を返す。
- source byte limit と statement output limit を分離する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp-parser test
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- cli.test.ts
```

## Task 3: Bounded asset lookup と parse metadata

対象:

- `packages/core/src/types.ts`
- `packages/core/src/assets.ts`
- `packages/core/src/parse.ts`
- `packages/core/src/index.ts`
- `packages/core/test/assets.test.ts`
- `packages/core/test/parsers.test.ts`

Red tests:

- assetId lookup が既定 5,000 files で停止する。
- cutoff 時に `visitedFileCount` と `cutoffReason` を返す。
- cutoff を通常の not-found として扱わない。
- direct path read は directory scan をしない。
- source size、bytes read、truncation が parse result まで残る。
- truncated `.a5er` を parse しない。
- truncated SQL は partial metadata を返す。

Implementation:

- `ReadAssetOptions` に `maxFiles` を追加する。
- metadata 付き lookup API を source of truth にする。
- `readA5sqlAsset()` は compatibility wrapper として残す。
- read / lookup metadata を `ParsedAssetResult` まで伝播する。
- `.a5er` truncation は fail closed にする。
- SQL / text は bounded partial result を許可する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp-core test
```

## Task 4: MCP trusted / untrusted contract

対象:

- `packages/cli/src/mcp/output-contract.ts`
- `packages/cli/src/mcp/tool-outputs.ts`
- `packages/cli/src/mcp/generation-tools.ts`
- `packages/cli/src/mcp/schema-compare/compare.ts`
- `packages/cli/src/mcp/tool-handler-utils.ts`
- `packages/cli/test/mcp-tools.test.ts`

Red tests:

- hostile table / column / live schema / tool input が `warnings`、`message`、`code`、`nextAction` に入らない。
- `warningDetails` が `untrustedPayloadFields` に入る。
- `warnings` は固定 code だけを含む。
- `content[0].text` と `structuredContent` が一致する。

Implementation:

- `warningDetails` と `connections` を untrusted field 候補へ追加する。
- dynamic warning を固定 code + detail object へ移行する。
- warning truncation も固定 code + numeric detail にする。
- draft disclosure を維持する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-tools.test.ts
```

## Task 5: Asset / connection handler と structured error

対象:

- `packages/cli/src/mcp/tool-handlers.ts`
- `packages/cli/src/mcp/tool-schemas.ts`
- `packages/cli/src/mcp/tool-handler-utils.ts`
- `packages/cli/test/mcp-asset-tools.test.ts`

Red tests:

- connection success / error response が untrusted contract を持つ。
- `read_a5sql_asset` が `maxBytes` を超えて再読込しない。
- lookup cutoff が `asset_lookup_truncated` になる。
- truncated `.a5er` が `source_truncated` になる。
- truncated SQL が partial result を返す。
- structured error が固定 `code`、`message`、`nextAction`、`retryable` を持つ。
- error response に絶対 path、assetId、入力値、生例外 message が漏れない。

Implementation:

- read / parse asset schema に `maxFiles` を追加する。
- masking 補助の全量再読込を削除する。
- lookup / source metadata を MCP response へ伝播する。
- `structuredErrorResult()` を追加する。
- connection response を `withUntrustedPayloadContract()` で包む。
- connection count semantics は 0.10.3 に残す。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-asset-tools.test.ts
```

## Task 6: Safe observability

対象:

- 新規 `packages/cli/src/mcp/observability.ts`
- `packages/cli/src/mcp/server.ts`
- `packages/cli/src/index.ts`
- observability tests

Red tests:

- 既定では log を出さない。
- opt-in 時だけ stderr sink へ JSON log を出す。
- password、token、connection string、SQL、table 名、assetId、絶対 path、生例外 message が log にない。
- tool name、latency、output size、fixed error code がある。
- 同一 process/key の同一 input は同じ hash になる。
- key が異なれば hash も異なる。

Implementation:

- `A5SQL_MCP_OBSERVABILITY=stderr` で opt-in にする。
- random process key を使った HMAC input hash を実装する。
- clock、key、sink を注入可能にする。
- tool handler を共通 observer wrapper で包む。
- transport error を固定 code と error 種別だけにする。
- stdout を JSON-RPC 専用に維持する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp test
```
