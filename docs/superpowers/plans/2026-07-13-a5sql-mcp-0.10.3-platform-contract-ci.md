## Task 1: Canonical text decoder

対象:

- `packages/core/src/text.ts`
- `packages/core/src/index.ts`
- `packages/core/src/types.ts`
- `packages/cli/src/text.ts`
- `packages/cli/src/index.ts`
- decoder fixture / tests

Red tests:

- CLI と core が同じ buffer から同じ text / encoding を返す。
- short BOM-less UTF-16LE を両方が UTF-16LE と判定する。
- tiny odd-position NUL binary を両方が binary と判定する。
- UTF-8 / UTF-8 BOM / UTF-16LE BOM / Shift_JIS / binary / empty が一致する。
- byte limit で multibyte 末尾が切れた UTF-8 を Shift_JIS と誤判定しない。
- `bytesRead` / `sizeBytes` / `truncated` が一致する。

Implementation:

- core decoder を唯一の実装にする。
- `decodeTextBuffer()` を options 付き public API にする。
- read API に `sizeBytes` を追加する。
- source truncation 時の strict UTF-8 streaming 判定を追加する。
- CLI decoder は core の compatibility facade にする。
- duplicated decoder logic を削除する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp-core test
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- cli.test.ts
```

## Task 2: Platform-aware path dedupe

対象:

- `packages/core/src/locations.ts`
- `packages/core/test/locations.test.ts`

Red tests:

- Linux では `/root/A5M2` と `/root/a5m2` を別候補として保持する。
- Windows では case だけ異なる path を dedupe する。
- relative / absolute 表記の同一 path は両 platform で dedupe する。
- defaults disabled の explicit root でも同じ規則を使う。

Implementation:

- `dedupePaths(items, platform)` にする。
- `win32` だけ resolved path を lowercase key 化する。
- Wine / Linux は case-sensitive key を維持する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp-core test -- locations.test.ts
```

## Task 3: Safe Laravel / SQLAlchemy model drafts

対象:

- `packages/cli/src/mcp/generation-tools.ts`
- 必要に応じた model generation helper module
- `packages/cli/test/mcp-tools.test.ts`
- dedicated syntax validation script / fixture

Red tests:

- PHP / Python reserved word が identifier に直接出ない。
- 先頭数字、空、記号、quote、backslash、control、非 ASCII 名が安全に処理される。
- 正規化後に衝突する table / column / relation 名が重複 identifier にならない。
- 元の physical table / column / foreign key 名が escaped string literal に保持される。
- SQLAlchemy の attribute と physical column 名が異なる場合に explicit column name が出る。
- draft disclosure と untrusted signal が維持される。
- generated PHP が `php -l`、Python が `python -m py_compile` を通る。

Implementation:

- PHP / Python identifier policy を別 helper にする。
- reserved word set と deterministic fallback / collision suffix を追加する。
- PHP / Python string literal encoder を追加する。
- relation method / key と model path も safe identifier を使う。
- file ごとに `syntaxValidation` metadata を返す。
- compile 用 synthetic hostile fixture と dedicated check command を追加する。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-tools.test.ts
rtk pnpm draft-syntax:check
```

## Task 4: Connection count metadata

対象:

- `packages/core/src/types.ts`
- `packages/core/src/connections.ts`
- `packages/core/src/index.ts`
- `packages/core/test/connections.test.ts`
- `packages/cli/src/mcp/tool-handlers.ts`
- `packages/cli/test/mcp-asset-tools.test.ts`

Red tests:

- scan 完了時に exact total / returned / truncated が一致する。
- client limit より候補が多い場合に total は全候補数、returned は limit になる。
- asset scan cutoff 時に total が `null` になる。
- unknown total を returned count で代用しない。
- `knownConnectionCount`、`totalConnectionCountIsExact`、visited / cutoff metadata が伝播する。
- existing compatibility API は connection array を返す。

Implementation:

- `listA5sqlConnectionsWithMetadata()` を source of truth にする。
- `searchA5sqlAssetsWithMetadata()` の cutoff metadata を使う。
- bounded scan 内では candidate count を最後まで数える。
- handler の `limit + 1` workaround を削除する。
- roots-required error variant の count fields も schema と整合させる。

Verification:

```bash
rtk pnpm --filter @takuyaw-w/a5sql-mcp-core test -- connections.test.ts
rtk pnpm --filter @takuyaw-w/a5sql-mcp test -- mcp-asset-tools.test.ts
```

## Task 5: Stable MCP outputSchema

対象:

- 新規 `packages/cli/src/mcp/output-schemas.ts`
- `packages/cli/src/mcp/server.ts`
- `packages/cli/src/mcp/tool-handler-utils.ts`

Red tests:

- stable 17 tool の `tools/list` に non-empty `outputSchema` がある。
- experimental draft 5 tool に stable output schema がない。
- output schema に tool ごとの required top-level field がある。
- success / error / unrecognized / truncated representative response が schema validation を通る。

Implementation:

- trusted / source / untrusted metadata fragment を定義する。
- structured error、unrecognized、paging / truncation fragment を定義する。
- stable tool ごとの output schema を定義する。
- nested payload の additive extension だけを許容する。
- `registerProfiledTool()` に tool-specific `outputSchema` を渡す。
- stable / draft classification と schema map の集合整合 test を追加する。

Verification:

## Task 6: CLI safe-by-default output

対象:

- `packages/cli/src/index.ts`
- CLI masking helper
- `packages/cli/test/cli.test.ts`
- README help examples

Red tests:

- direct SQL / text / A5:ER output の password、token、URL credential、connection string、private key が default stdout に出ない。
- output は parse 可能な JSON のままになる。
- non-secret metadata と parser structure は維持される。
- `--unsafe-raw-output` だけが raw value を返す。
- unsafe option は固定 stderr warning を出す。
- unknown direct-mode option を拒否する。
- MCP mode で unsafe direct-output option を受け付けない。

Implementation:

- parse-mode argument parser に `unsafeRawOutput` を追加する。
- recursive structured masking helper を追加する。
- `main()` の stdout 境界で default masking を適用する。
- help / docs に危険性と用途を記載する。
- `parseFile()` の internal raw contract は維持する。

Verification:
